# Serverless Architecture

sidebar_position: 4

## Overview

Serverless architecture allows building and running applications without managing infrastructure. GCP offers two primary serverless compute options: **Cloud Run** for containerized workloads and **Cloud Functions** for event-driven functions. Both scale automatically and you pay only for actual usage.

## When To Use

- Event-driven workloads (webhooks, file processing, scheduled tasks)
- APIs with variable, unpredictable traffic
- Microservices with independent scaling needs
- Rapid prototyping and experimentation
- Batch jobs and data processing pipelines
- workloads where cold start latency is acceptable

## When NOT To Use

- Long-running processes (> 10 minutes for Cloud Functions, > 60 min for Cloud Run)
- Consistent high-traffic workloads (may be cheaper with reserved instances)
- Applications requiring persistent connections or WebSockets at scale
- GPU/TPU intensive ML workloads
- Stateful workloads with complex local state
- When regulatory requirements mandate specific infrastructure control

## Core Concepts

### Cloud Run vs Cloud Functions

| Feature | Cloud Run | Cloud Functions |
|---------|-----------|-----------------|
| Runtime | Any container | Node.js, Python, Go, Java, Ruby, .NET |
| Cold starts | ~100-500ms | ~100-300ms |
| Max execution | 60 minutes | 60 seconds (1st gen), 60 min (2nd gen) |
| Concurrency | 1000 req/instance | 1 req/instance (1st gen), 80 (2nd gen) |
| Networking | VPC, Private Service Connect | Serverless VPC Access |
| Scaling | 0 to 1000 instances | 0 to 1000 instances |
| Min instances | Configurable | 0 (always cold) |

## Real Production Example

### Cloud Run - Next.js Web Application (Netflix-style)

Netflix serves video metadata through Cloud Run:

```typescript
// Next.js app with Cloud Run optimization
// package.json
{
  "scripts": {
    "start": "next start -p 8080",
    "build": "next build"
  }
}

// next.config.js
module.exports = {
  serverMinimumCacheTTL: 60 * 60, // 1 hour
  images: {
    loader: 'cloudinary',
    path: 'https://res.cloudinary.com/demo/image/fetch/'
  },
  // Optimize for serverless
  experimental: {
    serverActions: {
      allowedOrigins: ['devops-blog.example.com']
    }
  }
};

// app/api/content/route.ts
import { NextResponse } from 'next/server';

// Edge caching for content API
export const runtime = 'edge';
export const revalidate = 60; // ISR with 60s revalidation

export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);
  const category = searchParams.get('category') || 'all';
  const page = parseInt(searchParams.get('page') || '1');
  const limit = parseInt(searchParams.get('limit') || '20');

  // Cache in edge memory
  const cacheKey = `content:${category}:${page}:${limit}`;
  const cached = await caches.default.get(cacheKey);

  if (cached) {
    return cached;
  }

  // Fetch from database
  const content = await getContentByCategory(category, page, limit);

  const response = NextResponse.json({
    data: content.items,
    pagination: {
      page,
      limit,
      total: content.total,
      hasMore: page * limit < content.total
    }
  });

  // Cache for 5 minutes
  response.headers.set('Cache-Control', 'public, s-maxage=300, stale-while-revalidate=60');

  await caches.default.put(cacheKey, response.clone());

  return response;
}

async function getContentByCategory(category: string, page: number, limit: number) {
  // Cloud SQL query
  const offset = (page - 1) * limit;

  const [countResult, itemsResult] = await Promise.all([
    db.query('SELECT COUNT(*) as total FROM content WHERE category = ?', [category]),
    db.query('SELECT * FROM content WHERE category = ? LIMIT ? OFFSET ?', [category, limit, offset])
  ]);

  return {
    items: itemsResult.rows,
    total: countResult.rows[0].total
  };
}
```

### Cloud Functions - Event Processing (Stripe-style)

```typescript
// Cloud Function 2nd Gen - Image Processing Pipeline
import { Storage } from '@google-cloud/storage';
import { PubSub } from '@google-cloud/pubsub';
import { CloudEvent, CloudFunctionsEvent } from '@google-cloud/functions-framework';

const storage = new Storage();
const pubsub = new PubSub();

// Background function triggered by Cloud Storage
exportconst processUploadedImage = async (cloudEvent: CloudEvent<any>) => {
  const [metageneration, name, bucket] = [
    cloudEvent.data?.metageneration,
    cloudEvent.data?.name,
    cloudEvent.data?.bucket
  ];

  if (!name || !bucket) {
    console.error('Missing required event data');
    return;
  }

  // Only process images in the uploads folder
  if (!name.startsWith('uploads/')) {
    console.log('Skipping non-upload file:', name);
    return;
  }

  console.log(`Processing image: ${name} from bucket: ${bucket}`);

  try {
    // Get file metadata
    const file = storage.bucket(bucket).file(name);
    const [metadata] = await file.getMetadata();

    // Validate image type
    const contentType = metadata.contentType;
    if (!contentType?.startsWith('image/')) {
      console.error('Invalid file type:', contentType);
      throw new Error('Not an image file');
    }

    // Process image (generate thumbnails, optimize, etc.)
    const processedFiles = await processImage(file, name);

    // Publish completion event
    await pubsub.topic('image-processing-complete').publish({
      attributes: {
        originalFile: name,
        processedFiles: JSON.stringify(processedFiles)
      },
      messages: [{
        data: Buffer.from(JSON.stringify({
          originalFile: name,
          processedFiles,
          metadata: {
            size: metadata.size,
            contentType: metadata.contentType,
            created: metadata.timeCreated
          }
        }))
      }]
    });

    console.log('Image processed successfully:', processedFiles);
  } catch (error) {
    console.error('Image processing failed:', error);
    throw error; // Retry function
  }
};

// Scheduled cleanup function
exportconst scheduledCleanup = async (event: CloudFunctionsEvent) => {
  const bucket = storage.bucket('temp-uploads');

  const [files] = await bucket.getFiles({
    prefix: 'orphaned/',
    matchGlob: '**/*.tmp'
  });

  const thirtyDaysAgo = new Date();
  thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30);

  let deletedCount = 0;

  for (const file of files) {
    const [metadata] = await file.getMetadata();
    const created = new Date(metadata.timeCreated);

    if (created < thirtyDaysAgo) {
      await file.delete();
      deletedCount++;
      console.log('Deleted old temp file:', file.name);
    }
  }

  console.log(`Cleanup complete. Deleted ${deletedCount} files.`);
};

// HTTP function for webhooks
exportconst stripeWebhook = async (req: Express.Request, res: Express.Response) => {
  const sig = req.headers['stripe-signature'] as string;

  try {
    const event = stripe.webhooks.constructEvent(
      req.body,
      sig,
      process.env.STRIPE_WEBHOOK_SECRET!
    );

    switch (event.type) {
      case 'payment_intent.succeeded':
        await handlePaymentSuccess(event.data.object);
        break;
      case 'payment_intent.payment_failed':
        await handlePaymentFailure(event.data.object);
        break;
      case 'customer.subscription.created':
        await handleSubscriptionCreated(event.data.object);
        break;
    }

    res.json({ received: true });
  } catch (error) {
    console.error('Webhook error:', error);
    res.status(400).json({ error: 'Webhook error' });
  }
};
```

## Common Patterns / Anti-patterns

### Pattern 1: Serverless Batch Processing

```typescript
// Process items in parallel with controlled concurrency
exportconst processBatch = async (event: CloudEvent<any>) => {
  const items: string[] = JSON.parse(
    Buffer.from(event.data.message.data, 'base64').toString()
  );

  const concurrencyLimit = 10;
  const results: ProcessResult[] = [];

  // Process in chunks to avoid overwhelming resources
  for (let i = 0; i < items.length; i += concurrencyLimit) {
    const chunk = items.slice(i, i + concurrencyLimit);

    const chunkResults = await Promise.allSettled(
      chunk.map(item => processItem(item))
    );

    results.push(...chunkResults.map((result, idx) => ({
      item: chunk[idx],
      success: result.status === 'fulfilled',
      error: result.status === 'rejected' ? result.reason.message : null
    })));
  }

  // Store results
  await storeBatchResults(results);

  // Publish failures for retry
  const failed = results.filter(r => !r.success);
  if (failed.length > 0) {
    await pubsub.topic('batch-processing-failures').publish({
      messages: failed.map(f => ({
        data: Buffer.from(JSON.stringify(f))
      }))
    });
  }
};
```

### Pattern 2: Serverless GraphQL with Cloud Run

```typescript
// Apollo Server on Cloud Run
import { ApolloServer } from '@apollo/server';
import { expressMiddleware } from '@apollo/server/express4';
import express from 'express';

const typeDefs = `
  type Article {
    id: ID!
    title: String!
    content: String!
    author: Author!
    publishedAt: String!
  }

  type Author {
    id: ID!
    name: String!
    articles: [Article!]!
  }

  type Query {
    articles(limit: Int, offset: Int): [Article!]!
    article(id: ID!): Article
    authors: [Author!]!
  }
`;

const resolvers = {
  Query: {
    articles: async (_, { limit = 10, offset = 0 }) => {
      return db.query('SELECT * FROM articles LIMIT $1 OFFSET $2', [limit, offset]);
    },
    article: async (_, { id }) => {
      const result = await db.query('SELECT * FROM articles WHERE id = $1', [id]);
      return result.rows[0];
    },
    authors: async () => {
      return db.query('SELECT * FROM authors');
    }
  }
};

const server = new ApolloServer({
  typeDefs,
  resolvers,
  cache: 'bounded'
});

await server.start();

const app = express();

app.use('/graphql',
  cors<cors.CorsRequest>(),
  express.json(),
  expressMiddleware(server, {
    context: async ({ req }) => ({
      authHeader: req.headers.authorization
    })
  })
);

export default app;
```

### Anti-patterns

| Anti-pattern | Problem | Solution |
|-------------|---------|----------|
| Cold start on every request | High latency | Set min instances |
| Synchronous chain of functions | Timeout cascades | Use Pub/Sub for async |
| Stateful in-memory caching | Data loss on scale | Use Memorystore |
| Long-running connections | Connection limits | Use WebSockets via App Engine |
| Monolithic function | Slow builds, testing | Split into focused functions |
| No timeout configuration | Stuck instances | Set appropriate timeouts |

## Troubleshooting

### Common Serverless Issues

| Symptom | Likely Cause | Resolution |
|---------|--------------|------------|
| Cold start > 5s | Large image, no min instances | Reduce image size, set min instances |
| 503 on scaling | Startup probe too strict | Relax health check thresholds |
| Memory exhaustion | Memory limit too low | Increase memory or optimize usage |
| Timeout errors | Processing > timeout | Set higher timeout or optimize |
| Connection pool errors | Many concurrent requests | Use connection pooling or proxy |

## Interview Questions

### Q1: How do you handle state in a serverless application?
**Answer**: State should be externalized to persistent stores like Cloud SQL, Memorystore (Redis), or Cloud Storage. Avoid in-memory state since instances can scale to zero. For session data, use distributed caching with Memorystore. For workflow state, use Pub/Sub or Firestore.

### Q2: What strategies reduce cold start latency?
**Answer**: Set minimum instances to keep warm, use smaller container images, enable startup CPU boost, defer heavy imports, use languages with faster startup (Go, Java with GraalVM), and pre-warm connections on startup.

### Q3: When would you choose Cloud Functions over Cloud Run?
**Answer**: Cloud Functions is simpler for single-purpose event-driven tasks (file uploads, Pub/Sub handlers, webhooks). Cloud Run is better for APIs, web applications, and when you need more control over the runtime environment or need to run multiple processes in a container.

### Q4: How do you implement request deduplication in serverless?
**Answer**: Use idempotency keys with Cloud SQL or Firestore to track processed requests. Check if an idempotency key exists before processing. Set appropriate TTL on idempotency records. Cloud Run with min instances > 0 can share in-memory cache for short windows.

### Q5: What are the timeout considerations for serverless?
**Answer**: Cloud Functions 1st gen has 60s timeout (9min with billing), 2nd gen has 60min. Cloud Run has 60min max. For long operations, break into smaller steps with Pub/Sub chaining, or consider Cloud Workflows for orchestration.

## Cheat Sheet

```bash
# Deploy container to Cloud Run
gcloud run deploy content-api \
  --image gcr.io/${PROJECT_ID}/content-api:v1 \
  --region us-central1 \
  --min-instances 1 \
  --max-instances 100 \
  --concurrency 80 \
  --timeout 300s \
  --memory 1Gi \
  --cpu 2 \
  --vpc-connector my-connector \
  --set-env-vars NODE_ENV=production

# Deploy Cloud Function 2nd gen
gcloud functions deploy image-processor \
  --gen2 \
  --runtime nodejs18 \
  --region us-central1 \
  --source . \
  --entry-point processUploadedImage \
  --trigger-event-filters type=google.cloud.storage.object.v1.finalized \
  --trigger-event-filters bucket=${BUCKET_NAME} \
  --memory 512M \
  --timeout 300s

# Set secrets
echo -n "secret-value" | gcloud secrets create my-secret --data-file=-

# View metrics
gcloud monitoring metrics list | grep cloud_run
gcloud monitoring metrics list | grep cloud_functions

# IAM for service accounts
gcloud projects add-iam-policy-binding ${PROJECT_ID} \
  --member=serviceAccount:my-sa@${PROJECT_ID}.iam.gserviceaccount.com \
  --role=roles/cloudsql.client
```
