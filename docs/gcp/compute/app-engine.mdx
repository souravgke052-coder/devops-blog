# App Engine

## Overview

App Engine is Google's fully managed serverless platform for building and hosting applications. It abstracts away infrastructure management entirely—you upload your code and App Engine automatically handles deployment, scaling, load balancing, and health monitoring. App Engine supports multiple languages (Python, Java, Node.js, Go, PHP, Ruby, .NET) with built-in runtimes and also supports custom runtimes via Docker containers.

App Engine comes in two flavors: Standard and Flex (Flexible). Standard provides the most serverless experience with automatic scaling to zero and per-request pricing, while Flex provides more control with containerized deployments but doesn't scale to zero. Understanding when to use each is key to optimizing cost and performance.

App Engine excels at web applications, APIs, and backend services where you want to focus on code rather than infrastructure. It integrates with GCP services and provides built-in support for versioning, traffic splitting, and rollbacks.

## When To Use

- **Web applications**: Traditional web apps with request/response patterns
- **RESTful APIs**: Backend APIs for mobile or web frontends
- **Microservices**: Small, focused services in a larger architecture
- **Quick prototyping**: Fast deployment without infrastructure setup
- **Language-specific workloads**: Python, Node.js, Go, Java, PHP, Ruby
- **Standard environment**: When scaling to zero and per-request pricing are important
- **Traffic splitting**: A/B testing, canary deployments

## When NOT To Use

- **Containerized applications**: Use Cloud Run or GKE instead
- **Long-running processes**: App Engine has 60-second request timeout (Standard), 24 hours (Flex)
- **GPU/TPU workloads**: Not supported
- **Persistent processes**: Background tasks require Task Queue (Standard) or Cloud Tasks (Flex)
- **Binary/C++ workloads**: No native support
- **Complex networking**: App Engine is fairly locked down
- **Windows-specific applications**: Use Compute Engine or Cloud Run

## Architecture

**App Engine Architecture:**
```
App Engine Application
├── Service: default
│   ├── Version: v1 (traffic: 90%)
│   │   └── Instances: auto-scaling
│   └── Version: v2 (traffic: 10%)
│       └── Instances: auto-scaling
└── Service: api
    └── Version: v3
        └── Instances: manual-scaling
```

**Environment Comparison:**
| Feature | Standard | Flex |
|---------|----------|------|
| Scaling | To zero | Min 1 instance |
| Pricing | Per request | Per hour (instance) |
| SSH access | No | Yes |
| Dockerfile | No | Yes (custom runtime) |
| Request timeout | 60 seconds | 24 hours |
| Background threads | Limited | Full |
| Instance classes | F1-F4 | B2-B8 |

## Core Concepts

### Creating an App Engine Application

```bash
# Create App Engine application (must be done once per project)
gcloud app create --region=us-central

# Initialize app.yaml in current directory
gcloud app create --region=us-central --runtime=python312

# List regions
gcloud app regions list
```

### Deploying Applications

```bash
# Deploy to App Engine Standard
gcloud app deploy

# Deploy specific service
gcloud app deploy service.yaml

# Deploy with version label
gcloud app deploy --version=v2

# Deploy to specific service
gcloud app deploy --service=api

# Set traffic split during deployment
gcloud app deploy --promote

# Deploy without starting traffic
gcloud app deploy --no-promote

# View deployment progress
gcloud app deploy --verbosity=info
```

```yaml
# Python Standard Environment

entrypoint: gunicorn -b :$PORT main:app

  max_pending_latency: automatic

  ENV: production

- url: /.*
  script: auto
  secure: always
```

### Managing Services

```bash
# List services
gcloud app services list

# Describe service
gcloud app services describe default

# Set traffic for service
gcloud app services set-traffic default \
    --splits=v1=90,v2=10

# Split traffic by IP hash
gcloud app services set-traffic default \
    --splits=v1=80,v2=20 \
    --split-by=ip

# Split traffic by cookie
gcloud app services set-traffic default \
    --splits=v1=80,v2=20 \
    --split-by=cookie

# Delete service
gcloud app services delete api

# Browse app
gcloud app browse
gcloud app browse --service=api
gcloud app browse --version=v2
```

### Managing Versions

```bash
# List versions
gcloud app versions list

# List versions for specific service
gcloud app versions list --service=default

# Describe version
gcloud app versions describe v1 --service=default

# Traffic migration (move all traffic to version)
gcloud app versions migrate v2 --service=default

# Delete version
gcloud app versions delete v1 --service=default

# Get version logs
gcloud app versions logs v1 --service=default

# Split traffic to version
gcloud app services set-traffic default --splits=v1=100
```

### Instance Management

```bash
# List instances
gcloud app instances list

# List instances for service
gcloud app instances list --service=default

# Describe instance
gcloud app instances describe instance-id --service=default --version=v1

# Delete instance (Flex only)
gcloud app instances delete instance-id --service=default --version=v1

# SSH into instance (Flex only)
gcloud app ssh instance-id --service=default --version=v1
```

### Domain Mapping

```bash
# Map custom domain
gcloud app domain-mappings create example.com

# Map with SSL
gcloud app domain-mappings create api.example.com

# List domain mappings
gcloud app domain-mappings list

# Delete domain mapping
gcloud app domain-mappings delete example.com
```

### Quotas and Limits

| Resource | Limit |
|----------|-------|
| Services per app | 60 |
| Versions per service | 200 |
| Instances per version (Standard) | 100 |
| Request timeout (Standard) | 60 seconds |
| Request timeout (Flex) | 24 hours |
| Max request size | 32MB |
| Application memory (Standard) | 256MB-1GB |
| Application memory (Flex) | 1GB-8GB |

### Key Gotchas

- **Standard scales to zero**—first request after idle has cold start
- **60-second request timeout** in Standard—plan for long operations
- **No SSH in Standard**—debug with logs only
- **App Engine firewall** blocks all by default—explicitly allow
- **Custom domains need SSL**—automatic with managed certificates
- **Instance class determines resources**—choose based on memory/CPU needs
- **Flexible doesn't scale to zero**—always has at least 1 instance running

## Real Production Example

Production App Engine setup:

```bash
#!/bin/bash
set -e

PROJECT="my-project"
REGION="us-central"

# 1. Enable App Engine
gcloud services enable appengine.googleapis.com

# 2. Create App Engine application
gcloud app create --region=$REGION

# 3. Create app.yaml for web service
cat > app.yaml << 'EOF'
runtime: python312
entrypoint: gunicorn -b :$PORT main:app

instance_class: F2
automatic_scaling:
  min_instances: 2
  max_instances: 20
  target_cpu_utilization: 0.7
  min_pending_latency: 30ms
  max_pending_latency: automatic

handlers:
- url: /static
  static_dir: static
  secure: always
  
- url: /api/.*
  script: auto
  secure: always

- url: /.*
  script: auto
  secure: always
  redirect_http_response_code: 301

env_variables:
  ENV: "production"
  LOG_LEVEL: "info"
  DATABASE_URL: "postgresql://..."

builds:
- command: pip install -r requirements.txt
EOF

# 4. Create dispatch.yaml for routing
cat > dispatch.yaml << 'EOF'
dispatch:
  - url: "api.example.com/*"
    service: api
  - url: "admin.example.com/*"
    service: admin
  - url: "*/static/*"
    service: static-assets
EOF

# 5. Create cron.yaml for scheduled tasks
cat > cron.yaml << 'EOF'
cron:
- description: "daily cleanup job"
  url: /api/cleanup
  schedule: every day 02:00
  timezone: America/New_York

- description: "health check"
  url: /health
  schedule: every 5 minutes
EOF

# 6. Create queue.yaml for task queues
cat > queue.yaml << 'EOF'
queue:
- name: default
  rate: 100/m
  bucket_size: 50
  max_concurrent_requests: 100
  
- name: email-queue
  rate: 50/m
  target: api
EOF

# 7. Create index.yaml for Datastore (if using)
cat > index.yaml << 'EOF'
indexes:
- kind: Event
  properties:
  - name: date
    direction: desc
  - name: type
EOF

# 8. Create service accounts
gcloud iam service-accounts create app-engine-sa \
    --display-name="App Engine Service Account"

gcloud projects add-iam-policy-binding $PROJECT \
    --member="serviceAccount:app-engine-sa@${PROJECT}.iam.gserviceaccount.com" \
    --role="roles/cloudsql.client"

gcloud projects add-iam-policy-binding $PROJECT \
    --member="serviceAccount:app-engine-sa@${PROJECT}.iam.gserviceaccount.com" \
    --role="roles/storage.objectViewer"

# 9. Deploy application
gcloud app deploy --quiet

# 10. Create API service
mkdir -p api && cat > api/app.yaml << 'EOF'
runtime: python312
entrypoint: gunicorn -b :$PORT api:app

instance_class: F2
automatic_scaling:
  min_instances: 1
  max_instances: 10
  target_cpu_utilization: 0.6

env_variables:
  ENV: "production"

handlers:
- url: /.*
  script: auto
  secure: always
EOF

gcloud app deploy api/app.yaml --service=api

# 11. Deploy infrastructure files
gcloud app deploy cron.yaml
gcloud app deploy queue.yaml
gcloud app deploy dispatch.yaml

# 12. Set up custom domains
gcloud app domain-mappings create example.com
gcloud app domain-mappings create api.example.com

# 13. Split traffic (canary)
gcloud app services set-traffic default \
    --splits=v1=90,v2=10 \
    --split-by=cookie

# 14. View deployed version
gcloud app versions list

# 15. Browse application
gcloud app browse
```

```yaml
# Python Flask application
# main.py

    # Add readiness checks

@app.route('/api/data')

    # Example API endpoint
    return jsonify({'data': 'example'})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=int(os.environ.get('PORT', 8080)))
```

## Security

- Use service accounts with minimal permissions
- Enable HTTPS only (redirect_http_response_code: 301)
- Use Identity-Aware Proxy for admin access
- Implement proper authentication/authorization in code
- Use Secret Manager for sensitive data
- Enable Cloud Armor for WAF protection

## Cost Optimization

**App Engine Pricing:**
| Component | Cost |
|-----------|------|
| Instance hours (F1) | ~$0.05/hour |
| Instance hours (F2) | ~$0.10/hour |
| Instance hours (B2) | ~$0.10/hour |
| Requests | First 1M/month free, then ~$0.40/million |
| Custom domains | Free |

**Optimization:**
- Use automatic scaling with appropriate thresholds
- Set min_instances=1 only if cold starts are critical
- Use Standard for stateless request/response
- Delete unused versions
