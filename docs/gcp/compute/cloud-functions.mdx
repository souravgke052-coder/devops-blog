# Cloud Functions

## Overview

Cloud Functions is Google's serverless Functions-as-a-Service (FaaS) offering. It allows you to write single-purpose functions that respond to events without managing servers or containers. Cloud Functions automatically scales based on demand, charging only for the compute time when your function executes.

Cloud Functions is ideal for microservices, lightweight APIs, webhooks, automation scripts, and event-driven processing. It supports Node.js, Python, Go, Java, Ruby, PHP, and .NET. Unlike Cloud Run (which runs containers), Cloud Functions uses language-specific runtimes and has a simpler deployment model but less flexibility.

Key features include event-driven execution (triggered by Cloud Storage, Pub/Sub, Firestore, HTTP, etc.), automatic scaling, per-millisecond billing, built-in logging and monitoring, VPC support, and integration with other GCP services.

## When To Use

- **Event-driven processing**: React to Cloud Storage, Pub/Sub, or Firestore events
- **Lightweight APIs**: Simple HTTP endpoints
- **Webhooks**: Receive and process webhooks
- **Automation scripts**: Scheduled or triggered automation
- **Data transformation**: Process streams or files on upload
- **Chatbots and assistants**: Respond to messaging platform events
- **Prototyping**: Quick deployment of simple functions

## When NOT To Use

- **Containerized applications**: Use Cloud Run
- **Complex applications**: Multiple endpoints, long-running processes
- **GPU/TPU workloads**: Not supported
- **Heavy dependencies**: Cloud Run gives more control
- **Stateful applications**: Functions should be stateless
- **Complex routing**: API Gateway or Cloud Run for advanced routing
- **Long-running tasks**: Functions have timeout limits (up to 60 minutes with 2nd gen)

## Architecture

**Cloud Functions Gen 1 vs Gen 2:**

| Feature | Gen 1 | Gen 2 |
|---------|-------|-------|
| Max timeout | 9 minutes (HTTP: 60 min) | 60 minutes |
| Max memory | 8GB | 16GB |
| CPU | Tied to memory | Separate allocation |
| Concurrency | 1 request/instance | 80 requests/instance |
| Min instances | N/A | Configurable |
| Max instances | 1000 | 1000 |
| Triggers | Cloud Events v1 | Cloud Events v1 |
| Networking | Basic | VPC, Direct VPC |

**Event Triggers:**
```
Cloud Storage Object Create → Trigger Function → Process file
Pub/Sub Message → Trigger Function → Process message
Firestore Create/Update → Trigger Function → React to data change
HTTP Request → Trigger Function → Handle request
Cloud Scheduler → Trigger Function → Scheduled task
```

## Core Concepts

### Creating Functions

```bash
# Deploy HTTP function (Node.js)
gcloud functions deploy my-http-function \
    --runtime=nodejs20 \
    --trigger-http \
    --allow-unauthenticated \
    --region=us-central1 \
    --entry-point=handleRequest \
    --memory=256MB \
    --timeout=60s

# Deploy HTTP function (Python)
gcloud functions deploy my-python-function \
    --runtime=python312 \
    --trigger-http \
    --allow-unauthenticated \
    --region=us-central1 \
    --entry-point=handle_request

# Deploy with specific settings
gcloud functions deploy my-function \
    --runtime=nodejs20 \
    --trigger-http \
    --region=us-central1 \
    --memory=512MB \
    --timeout=120s \
    --max-instances=10 \
    --min-instances=1 \
    --concurrency=100 \
    --cpu=1 \
    --ingress-settings=internal-and-cloud-load-balancing \
    --vpc-connector=my-vpc-connector \
    --vpc-egress=all-traffic \
    --set-env-vars="ENV=production" \
    --add-env-vars="VERSION=1.0" \
    --remove-env-vars="DEBUG" \
    --service-account=my-sa@my-project.iam.gserviceaccount.com

# Deploy Cloud Storage trigger
gcloud functions deploy my-storage-function \
    --runtime=nodejs20 \
    --trigger-bucket=my-bucket \
    --trigger-event-filters="type=google.cloud.storage.object.v1.finalized" \
    --region=us-central1

# Deploy Pub/Sub trigger
gcloud functions deploy my-pubsub-function \
    --runtime=nodejs20 \
    --trigger-topic=my-topic \
    --trigger-event-filters="type=google.cloud.pubsub.topic.v1.messagePublished" \
    --region=us-central1

# Deploy with retry on failure
gcloud functions deploy my-function \
    --runtime=nodejs20 \
    --trigger-http \
    --region=us-central1 \
    --retry

# Deploy with Cloud Scheduler (HTTP trigger)
gcloud functions deploy my-scheduled-function \
    --runtime=nodejs20 \
    --trigger-http \
    --region=us-central1

gcloud scheduler jobs create http my-scheduled-job \
    --location=us-central1 \
    --schedule="0 2 * * *" \
    --uri=$(gcloud functions describe my-scheduled-function --region=us-central1 --format="value(httpsTrigger.url)") \
    --http-method=POST
```

### IAM and Security

```bash
# Allow unauthenticated access (public)
gcloud functions add-iam-policy-binding my-function \
    --region=us-central1 \
    --member="allUsers" \
    --role="roles/cloudfunctions.invoker"

# Allow specific user
gcloud functions add-iam-policy-binding my-function \
    --region=us-central1 \
    --member="user:alice@company.com" \
    --role="roles/cloudfunctions.invoker"

# Allow service account
gcloud functions add-iam-policy-binding my-function \
    --region=us-central1 \
    --member="serviceAccount:sa@project.iam.gserviceaccount.com" \
    --role="roles/cloudfunctions.invoker"

# Remove public access
gcloud functions remove-iam-policy-binding my-function \
    --region=us-central1 \
    --member="allUsers" \
    --role="roles/cloudfunctions.invoker"

# Get IAM policy
gcloud functions get-iam-policy my-function --region=us-central1
```

### Quotas and Limits

| Resource | Limit |
|----------|-------|
| Max timeout (Gen 1 HTTP) | 60 minutes |
| Max timeout (Gen 2) | 60 minutes |
| Max memory (Gen 1) | 8GB |
| Max memory (Gen 2) | 16GB |
| Max concurrent invocations | 1,000 |
| Max instances | 1,000 |
| Request body size | 10MB |
| Max deployment size | 100MB (compressed) |

### Key Gotchas

- **Functions are stateless**—no persistent local storage
- **Cold starts**: Gen 2 with min-instances=0 may have cold starts
- **Timeout applies per invocation**—long tasks need chunking
- **Gen 2 concurrency**: 80 concurrent requests per instance
- **Retries are per-invocation**—enable retry flag for event triggers
- **Environment variables limited**—use Secret Manager for secrets
- **No concurrent updates**—deployment is serialized

## Real Production Example

Production Cloud Functions setup:

```bash
#!/bin/bash
set -e

PROJECT="my-project"
REGION="us-central1"

# 1. Enable required APIs
gcloud services enable cloudfunctions.googleapis.com \
    cloudbuild.googleapis.com \
    eventarc.googleapis.com \
    pubsub.googleapis.com \
    storage.googleapis.com

# 2. Create service account
gcloud iam service-accounts create function-sa \
    --display-name="Cloud Function Service Account"

gcloud projects add-iam-policy-binding $PROJECT \
    --member="serviceAccount:function-sa@${PROJECT}.iam.gserviceaccount.com" \
    --role="roles/storage.objectViewer"

gcloud projects add-iam-policy-binding $PROJECT \
    --member="serviceAccount:function-sa@${PROJECT}.iam.gserviceaccount.com" \
    --role="roles/cloudsql.client"

gcloud projects add-iam-policy-binding $PROJECT \
    --member="serviceAccount:function-sa@${PROJECT}.iam.gserviceaccount.com" \
    --role="roles/pubsub.publisher"

# 3. Create HTTP API function
cat > index.js << 'EOF'
const express = require('express');
const app = express();

app.use(express.json());

// Health check
app.get('/health', (req, res) => {
  res.json({ status: 'healthy', timestamp: new Date().toISOString() });
});

// Main API endpoint
app.post('/api/process', async (req, res) => {
  try {
    const { data } = req.body;
    // Process data
    const result = {
      received: data,
      processed: true,
      timestamp: new Date().toISOString()
    };
    res.json(result);
  } catch (error) {
    console.error('Error:', error);
    res.status(500).json({ error: 'Processing failed' });
  }
});

exports.apiHandler = app;
EOF

cat > package.json << 'EOF'
{
  "name": "api-function",
  "version": "1.0.0",
  "dependencies": {
    "express": "^4.18.0"
  }
}
EOF

gcloud functions deploy api-function \
    --runtime=nodejs20 \
    --gen2 \
    --trigger-http \
    --region=$REGION \
    --entry-point=apiHandler \
    --memory=512MB \
    --cpu=1 \
    --timeout=60s \
    --min-instances=1 \
    --max-instances=100 \
    --concurrency=80 \
    --service-account=function-sa@${PROJECT}.iam.gserviceaccount.com \
    --set-env-vars="NODE_ENV=production" \
    --ingress-settings=internal-and-cloud-load-balancing \
    --vpc-connector=my-vpc-connector \
    --vpc-egress=all-traffic

gcloud functions add-iam-policy-binding api-function \
    --region=$REGION \
    --member="allUsers" \
    --role="roles/cloudfunctions.invoker"

# 4. Create Cloud Storage trigger for image processing
cat > image-processor/index.js << 'EOF'
exports.processImage = async (event, context) => {
  const file = event;
  const bucket = file.bucket;
  const filename = file.name;
  
  console.log(`Processing ${filename} from ${bucket}`);
  
  // Only process images
  if (!filename.match(/\.(jpg|png|gif)$/i)) {
    console.log('Skipping non-image file');
    return;
  }
  
  // Process image (placeholder)
  console.log(`Image processed: ${filename}`);
};
EOF

gcloud functions deploy image-processor \
    --runtime=nodejs20 \
    --gen2 \
    --trigger-event-filters="type=google.cloud.storage.object.v1.finalized" \
    --trigger-event-filters="bucket=${PROJECT}-images" \
    --region=$REGION \
    --entry-point=processImage \
    --memory=512MB \
    --timeout=120s \
    --service-account=function-sa@${PROJECT}.iam.gserviceaccount.com \
    --retry

# 5. Create Pub/Sub trigger for message processing
cat > message-handler/index.js << 'EOF'
exports.handleMessage = async (event) => {
  const message = Buffer.from(event.data.data, 'base64').toString();
  const attrs = event.attributes;
  
  console.log(`Processing message: ${message}`);
  console.log(`Attributes:`, attrs);
  
  // Process message
  return { processed: true };
};
EOF

gcloud functions deploy message-handler \
    --runtime=nodejs20 \
    --gen2 \
    --trigger-event-filters="type=google.cloud.pubsub.topic.v1.messagePublished" \
    --trigger-event-filters="topic=my-topic" \
    --region=$REGION \
    --entry-point=handleMessage \
    --memory=256MB \
    --service-account=function-sa@${PROJECT}.iam.gserviceaccount.com \
    --retry

# 6. Create scheduled function
cat > scheduled-task/index.js << 'EOF'
exports.scheduledTask = async (event) => {
  console.log('Running scheduled task');
  // Perform cleanup, aggregations, etc.
  return { success: true };
};
EOF

gcloud functions deploy scheduled-task \
    --runtime=nodejs20 \
    --gen2 \
    --trigger-http \
    --region=$REGION \
    --entry-point=scheduledTask \
    --memory=256MB

# Create Cloud Scheduler job
gcloud scheduler jobs create http scheduled-job \
    --location=$REGION \
    --schedule="0 3 * * *" \
    --uri=$(gcloud functions describe scheduled-task --region=$REGION --format="value(httpsTrigger.url)") \
    --http-method=POST \
    --oidc-service-account-email=function-sa@${PROJECT}.iam.gserviceaccount.com

# 7. Test functions
gcloud functions call api-function --region=$REGION --data='{"data":"test"}'
gcloud functions call health-check --region=$REGION

# 8. View logs
gcloud functions logs read api-function --region=$REGION --limit=50
```

## Security

- Use service accounts with minimal permissions
- Don't expose sensitive data in environment variables—use Secret Manager
- Set ingress settings appropriately (internal-only if needed)
- Use VPC connector for private network access
- Enable retry for event triggers to handle transient failures

## Cost Optimization

**Cloud Functions Pricing:**
- Invocations: First 2 million/month free, then ~$0.40/million
- CPU: $0.00001600/vCPU-second (Gen 2 only when executing)
- Memory: $0.00000250/GiB-second
- Outbound egress: Standard rates

**Optimization:**
- Use Gen 2 for better concurrency (80 vs 1)
- Set max-instances to prevent runaway costs
- Use min-instances=0 for infrequently called functions
- Optimize function execution time
