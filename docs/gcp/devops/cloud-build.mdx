# Cloud Build

## Overview

Google Cloud Build is a fully managed CI/CD platform that executes builds on Google Cloud infrastructure. It integrates with source repositories, artifact registries, and deployment targets. Cloud Build supports container builds, non-container builds, and parallel task execution with pay-per-second pricing.

## When To Use

- Building container images
- CI/CD pipeline automation
- Deploying to GKE, Cloud Run, or Firebase
- Multi-step build pipelines
- Testing and quality checks
- Binary compilation

## When NOT To Use

- Simple file copying (use gsutil)
- Long-running massive builds (consider self-hosted)
- Windows container builds (not supported)
- Very high-frequency builds (< 1 min intervals)

## Core Concepts

### Build Trigger
Automates builds based on source events (push, PR, etc.).

### Build Config
YAML file defining build steps, artifacts, and options.

### Build Step
Individual action in a build (run Docker container).

### Builder
Pre-built containers for common tasks (docker, gcloud, npm, etc.).

### Worker Pool
Dedicated infrastructure for sensitive builds.

# Terraform configuration for Cloud Build setup

# Enable APIs
resource "google_project_service" "cloudbuild" {
  service = "cloudbuild.googleapis.com"
}

resource "google_project_service" "artifact" {
  service = "artifactregistry.googleapis.com"
}

resource "google_project_service" "run" {
  service = "run.googleapis.com"
}

# Create artifact registry repository
resource "google_artifact_registry_repository" "app" {
  location      = "us-central1"
  repository_id = "app"
  format       = "DOCKER"
}

# Create service account for Cloud Build
resource "google_service_account" "cloudbuild_sa" {
  account_id = "cloudbuild-sa"
  display_name = "Cloud Build Service Account"
}

# Grant necessary roles to Cloud Build SA
resource "google_project_iam_member" "build_roles" {
  count = length([
    "roles/container.developer",
    "roles/run.admin",
    "roles/artifactregistry.writer",
    "roles/secretmanager.secretAccessor",
    "roles/logging.logWriter",
    "roles/storage.objectViewer",
  ])
  
  project = var.project
  role    = element([
    "roles/container.developer",
    "roles/run.admin",
    "roles/artifactregistry.writer",
    "roles/secretmanager.secretAccessor",
    "roles/logging.logWriter",
    "roles/storage.objectViewer",
  ], count.index)
  member  = "serviceAccount:${google_service_account.cloudbuild_sa.email}"
}

# Build trigger for main branch
resource "google_cloudbuild_trigger" "main_branch_trigger" {
  name        = "main-branch-build"
  description = "Build and deploy on main branch push"
  
  github {
    owner = "my-org"
    repo  = "my-app"
    push {
      branch = "^main$"
    }
  }
  
  service_account = google_service_account.cloudbuild_sa.id
  
  filename = "cloudbuild.yaml"
  
  substitutions = {
    _REGION     = "us-central1"
    _SERVICE_NAME = "my-app"
  }
  
  tags = ["deploy", "production"]
}

# Build trigger for PR
resource "google_cloudbuild_trigger" "pr_trigger" {
  name        = "pr-build"
  description = "Build and test on pull request"
  
  github {
    owner = "my-org"
    repo  = "my-app"
    pull_request {
      branch = "^main$"
    }
  }
  
  service_account = google_service_account.cloudbuild_sa.id
  
  filename = "cloudbuild.yaml"
  
  substitutions = {
    _REGION     = "us-central1"
    _SERVICE_NAME = "my-app"
    _BUILD_TYPE = "pr"
  }

}

### cloudbuild.yaml

```yaml
# Cloud Build Configuration
# Build, test, and deploy application

substitutions:
  _REGION: us-central1
  _SERVICE_NAME: my-app
  _IMAGE: $_REGION-docker.pkg.dev/$PROJECT_ID/app/$_SERVICE_NAME

options:
  logging: GCS_ONLY
  machineType: E2_HIGHCPU_8
  dynamic_substitutions: true
  volumes:
    - name: 'npm'
      path: '/root/.npm'
    - name: 'cache'
      path: '/root/.cache'

steps:
  # Prepare environment
  - name: 'gcr.io/cloud-builders/docker'
    id: 'docker-auth'
    entrypoint: 'bash'
    args:
      - '-c'
      - |
        gcloud auth configure-docker $_REGION-docker.pkg.dev

  # Run tests
  - name: 'node:18'
    id: 'test'
    entrypoint: 'npm'
    args:
      - 'ci'
    env:
      - 'CI=true'
    volumes:
      - name: 'npm'
        path: '/root/.npm'
      - name: 'cache'
        path: '/root/.cache'

  - name: 'node:18'
    id: 'lint'
    entrypoint: 'npm'
    args:
      - 'run'
      - 'lint'
    waitFor: ['-']  # Run in parallel with test
    env:
      - 'CI=true'

  - name: 'node:18'
    id: 'unit-tests'
    entrypoint: 'npm'
    args:
      - 'run'
      - 'test:unit'
    waitFor: ['-']
    env:
      - 'CI=true'

  - name: 'node:18'
    id: 'integration-tests'
    entrypoint: 'npm'
    args:
      - 'run'
      - 'test:integration'
    waitFor: ['test', 'lint']
    env:
      - 'CI=true'

  # Build container image
  - name: 'gcr.io/cloud-builders/docker'
    id: 'build'
    args:
      - 'build'
      - '-t'
      - '$_IMAGE:$COMMIT_SHA'
      - '-t'
      - '$_IMAGE:latest'
      - '--cache-from'
      - '$_IMAGE:latest'
      - '--build-arg'
      - 'BUILD_VERSION=$COMMIT_SHA'
      - '.'
    waitFor: ['-']  # Run in parallel with tests

  # Security scan
  - name: 'gcr.io/cloud-builders/gcloud'
    id: 'container-analysis'
    entrypoint: 'bash'
    args:
      - '-c'
      - |
        gcloud artifacts docker describe $_IMAGE:$COMMIT_SHA --show-discovery-results || echo "No vulnerabilities found"
    waitFor: ['build']

  # Push images
  - name: 'gcr.io/cloud-builders/docker'
    id: 'push-sha'
    args:
      - 'push'
      - '$_IMAGE:$COMMIT_SHA'
    waitFor: ['container-analysis']

  - name: 'gcr.io/cloud-builders/docker'
    id: 'push-latest'
    args:
      - 'push'
      - '$_IMAGE:latest'
    waitFor: ['push-sha']

  # Deploy to staging (PR builds)
  - name: 'gcr.io/google.com/cloudsdktool/cloud-sdk'
    id: 'deploy-staging'
    entrypoint: 'gcloud'
    args:
      - 'run'
      - 'deploy'
      - '$_SERVICE_NAME-staging'
      - '--image=$_IMAGE:$COMMIT_SHA'
      - '--region=$_REGION'
      - '--tag=staging'
      - '--allow-unauthenticated'
    waitFor: ['push-sha']
    condition: '$_BUILD_TYPE == "pr"'

  # Deploy to production (main/release builds)
  - name: 'gcr.io/google.com/cloudsdktool/cloud-sdk'
    id: 'deploy-production'
    entrypoint: 'gcloud'
    args:
      - 'run'
      - 'deploy'
      - '$_SERVICE_NAME'
      - '--image=$_IMAGE:$COMMIT_SHA'
      - '--region=$_REGION'
      - '--allow-unauthenticated'
      - '--tag=$COMMIT_SHA'
    waitFor: ['push-sha']
    condition: '$_BUILD_TYPE != "pr"'

  # Notify completion (for main/release)
  - name: 'gcr.io/cloud-builders/gcloud'
    id: 'notify-success'
    entrypoint: 'bash'
    args:
      - '-c'
      - |
        echo "Build completed successfully"
        echo "Commit: $COMMIT_SHA"
        echo "Image: $_IMAGE:$COMMIT_SHA"
    waitFor: ['deploy-production', 'deploy-staging']
    condition: '$_BUILD_TYPE != "pr"'

images:
  - '$_IMAGE:$COMMIT_SHA'
  - '$_IMAGE:latest'

tags:
  - '$_SERVICE_NAME'
  - '$_BUILD_TYPE'

timeout: '1800s'
```

## CLI Examples

```bash
# Submit build from local Dockerfile
gcloud builds submit --tag us-central1-docker.pkg.dev/my-project/my-repo/my-image:tag .

# Submit build with cloudbuild.yaml
gcloud builds submit --config=cloudbuild.yaml .

# Submit build with substitutions
gcloud builds submit --config=cloudbuild.yaml \
  --substitutions=_REGION=us-east1,_ENV=prod .

# Run build without source (from config only)
gcloud builds submit --config=cloudbuild.yaml --no-source

# List builds
gcloud builds list --limit=50

# Describe build
gcloud builds describe BUILD_ID

# Cancel build
gcloud builds cancel BUILD_ID

# Create trigger
gcloud builds triggers create github \
  --repo-name=my-repo \
  --repo-owner=my-org \
  --branch-pattern="^main$" \
  --build-config=cloudbuild.yaml

# Create trigger with substitution
gcloud builds triggers create github \
  --repo-name=my-repo \
  --repo-owner=my-org \
  --tag-pattern="^v[0-9.]+$" \
  --build-config=cloudbuild.yaml \
  --substitutions=_VERSION='$(tag)'

# Delete trigger
gcloud builds triggers delete TRIGGER_ID

# View build logs
gcloud builds log BUILD_ID

# Use worker pool
gcloud builds submit --config=cloudbuild.yaml \
  --worker-pool=projects/my-project/locations/us-central1/workerPools/my-pool
```

## Interview Questions

**Q: How does Cloud Build caching work?**
A: Cloud Build caches layer by storing intermediate images. Use `--cache-from` to pull previous image layers. In Dockerfile, use multi-stage builds. Cloud Build also caches steps by default when using `waitFor: ['-']` and the same step runs again.

**Q: What is the difference between standard and private worker pools?**
A: Standard pools use shared infrastructure. Private worker pools provide dedicated machines in your VPC for better security, network access, and compliance requirements. Private pools also support custom machine types.

**Q: How do you handle secrets in Cloud Build?**
A: Store secrets in Secret Manager with the Cloud Build SA as a principal. In cloudbuild.yaml, retrieve secrets using `gcloud secrets versions access`. Use `$$` to prevent echo expansion.

**Q: What triggers are available in Cloud Build?**
A: GitHub/GitHub Enterprise pushes and pull requests, Cloud Source Repository pushes, Pub/Sub triggers, Manual triggers (via API), and scheduled triggers via Cloud Scheduler.

**Q: How do you optimize Cloud Build performance?**
A: Use `--cache-from` for Docker layers, parallelize independent steps with `waitFor: ['-']`, use smaller machine types for simple builds, implement Docker layer caching in Dockerfile, and use remote build caching with `--disk-size` optimization.
