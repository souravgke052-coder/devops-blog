# Error Reporting

## Overview

Google Cloud Error Reporting automatically groups and counts errors from cloud services and application logs. It provides real-time error visibility, stack traces, and integrates with incident management. Error Reporting supports multiple languages and frameworks out of the box.

## When To Use

- Application error tracking
- Production issue identification
- Error trend analysis
- Incident correlation
- Stack trace analysis

## When NOT To Use

- Log aggregation (use Cloud Logging)
- Performance monitoring (use Cloud Monitoring)
- Distributed tracing (use Cloud Trace)

## Core Concepts

### Error Group
Collection of similar errors based on stack trace fingerprint.

### Error Occurrence
Individual error event with timestamp and details.

### Exception
Error with stack trace and location information.

### Impact
Count of affected users and error frequency.

### Application Integration

#### Python (Flask)

```python
# pip install google-cloud-error-reporting
# google-cloud-error-reporting-lib or use Google Cloud Logging handler

import logging
from google.cloud import logging as cloud_logging
from google.cloud.logging_v2.handlers import CloudLoggingHandler
from google.cloud.error_reporting import setup_logging

# Configure error reporting
setup_logging(level=logging.ERROR)

# Or manually with Cloud Logging
client = cloud_logging.Client()
handler = CloudLoggingHandler(client)
logging.root.addHandler(handler)
logging.root.setLevel(logging.ERROR)

# Log exceptions
try:
    risky_operation()
except Exception as e:
    logging.error(
        f"Exception occurred: {str(e)}",
        exc_info=True,  # Include stack trace
        extra={
            "context": {
                "user_id": get_current_user_id(),
                "request_id": get_request_id(),
            }
        }
    )
```

#### Node.js (Express)

```javascript
// npm install @google-cloud/error-reporting

const { ErrorReporting } = require('@google-cloud/error-reporting');

// Initialize error reporting
const errors = new ErrorReporting({
    projectId: 'my-project',
    serviceContext: {
        service: 'my-service',
        version: '1.0.0'
    }
});

// Use as error handler middleware
const express = require('express');
const app = express();

app.use(errors.express);

app.get('/error', (req, res) => {
    throw new Error('Intentional error for testing');
});

// Or manually report errors
app.use((err, req, res, next) => {
    errors.report(err, {
        user: req.user?.id,
        requestId: req.headers['x-request-id']
    });
    next(err);
});
```

#### Go

```go
package main

import (
    "contrib.go.opencensus.io/exporter/stackdriver"
    "go.opencensus.io/trace"
)

// For error reporting with Cloud Logging and Error Reporting
import (
    "cloud.google.com/go/errorreporting"
    "cloud.google.com/go/errorreporting/apiv1beta1"
)

// Initialize error reporting
client, err := errorreporting.NewClient(ctx, "my-project",
    errorreporting.Config{
        ServiceName:    "my-service",
        ServiceVersion: "1.0.0",
    },
)
defer client.Close()

// Report errors
client.Report(errorreporting.Entry{
    Error: err,
    Context: map[string]string{
        "user": "user-id",
    },
})
```

## Interview Questions

**Q: How does Error Reporting group errors?**
A: Error Reporting uses stack trace fingerprints to group similar errors. Functions and line numbers in the top of the stack trace determine the group. Similar errors from different users are grouped together.

**Q: What is the difference between Error Reporting and Cloud Logging?**
A: Cloud Logging stores all log entries. Error Reporting automatically extracts and analyzes errors, providing grouping, trends, and stack traces. Error Reporting reads from Cloud Logging logs with ERROR or CRITICAL severity.

**Q: How do you integrate Error Reporting with custom applications?**
A: Use language-specific Error Reporting libraries (Python, Node.js, Java, Go). Install SDK, configure service context, and call report() on exceptions. For custom logs, ensure ERROR-level logs are written to Cloud Logging.

**Q: What triggers a new error group?**
A: A new error group is created when an error's stack trace fingerprint doesn't match existing groups. The fingerprint is based on the error's type, message, and top of stack trace.

**Q: How does Error Reporting handle sensitive data?**
A: Stack traces may contain file paths and function names. User data should be minimized. Consider sanitizing error messages before logging to prevent exposing sensitive data.
