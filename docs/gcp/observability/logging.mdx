# Cloud Logging

## Overview

Google Cloud Logging (formerly Stackdriver) provides centralized logging for GCP and hybrid environments. It ingests logs from all GCP services, VMs, and custom applications. Logs are indexed, searchable, and can be exported to Cloud Storage, BigQuery, or Pub/Sub for analysis and archival.

## When To Use

- Application debugging and troubleshooting
- Audit compliance logging
- Security monitoring
- Log-based metrics
- Incident investigation
- Compliance archiving

## When NOT To Use

- High-frequency trading data (use specialized solutions)
- Real-time stream processing (use Dataflow)
- Long-term cold storage (use Cloud Storage directly)

## Architecture

```
│                     Cloud Logging                            │
├─────────────────────────────────────────────────────────────┤
│  Log Router:                                                │

│  Log Storage:                                              │

  Pricing:
  ├── First 50 GB/month: Free
  └── $0.005/GB after free tier
└─────────────────────────────────────────────────────────────┘
```

## Core Concepts

### Log Entry
Individual log record with timestamp, severity, payload, and resource metadata.

### Log Sink
Destination for log routing with filter-based routing.

### Log Router
Ingests and routes logs based on sink configuration.

### Log Metric
Metric derived from log entries via filtering.

### Log Bucket
Storage container for log categories with retention settings.

### Audit Log
GCP service activity logs for compliance.

## Real Production Example

**Log Management Infrastructure**

```hcl
# Terraform configuration for Cloud Logging

# Create log sink to Cloud Storage for archival
resource "google_logging_project_sink" "storage_sink" {
  name        = "storage-archive-sink"
  destination = "storage.googleapis.com/${google_storage_bucket.audit_archive.name}"
  
  filter = <<-EOT
    logName="projects/${var.project}/logs/cloudaudit.googleapis.com%2Factivity" OR
    logName="projects/${var.project}/logs/cloudaudit.googleapis.com%2Fdata_access"
  EOT
  
  unique_writer_identity = true
}

# Storage bucket for logs
resource "google_storage_bucket" "audit_archive" {
  name          = "${var.project}-audit-logs"
  location      = "US"
  storage_class = "STANDARD"
  
  uniform_bucket_level_access = true
  
  lifecycle_rule {
    condition {
      age = 365  # Retain for 1 year
    }
    action {
      type = "SetStorageClass"
      storage_class = "NEARLINE"
    }
  }
  
  lifecycle_rule {
    condition {
      age = 1825  # Delete after 5 years
    }
    action {
      type = "Delete"
    }
  }
}

# Create sink to BigQuery for analytics
resource "google_logging_project_sink" "bigquery_sink" {
  name        = "bigquery-analytics-sink"
  destination = "bigquery.googleapis.com/projects/${var.project}/datasets/${google_bigquery_dataset.analytics.dataset_id}"
  
  filter = <<-EOT
    resource.type="cloud_run_revision" AND
    severity>=WARNING
  EOT
  
  unique_writer_identity = true
}

# BigQuery dataset for log analytics
resource "google_bigquery_dataset" "analytics" {
  dataset_id    = "logs_analytics"
  friendly_name = "Logs Analytics"
  description   = "Log data for analytics"
  location      = "US"
  
  default_table_expiration_ms = 2592000000  # 30 days
}

# Create sink to Pub/Sub for real-time processing
resource "google_logging_project_sink" "pubsub_sink" {
  name        = "pubsub-realtime-sink"
  destination = "pubsub.googleapis.com/${var.project}/${google_pubsub_topic.logs.id}"
  
  filter = <<-EOT
    resource.type="k8s_container" AND
    severity>=ERROR
  EOT
  
  unique_writer_identity = true
}

# Pub/Sub topic for real-time logs
resource "google_pubsub_topic" "logs" {
  name = "realtime-logs"
}

# Log metrics
resource "google_logging_metric" "error_count" {
  name   = "error_count"
  filter = "severity>=ERROR"
  
  metric_descriptor {
    metric_kind = "DELTA"
    value_type  = "INT64"
    unit        = "1"
  }
}

resource "google_logging_metric" "api_latency" {
  name   = "api_latency"
  filter = <<-EOT
    resource.type="cloud_run_revision" AND
    metric.type="run.googleapis.com/request_latencies"
  EOT
  
  metric_descriptor {
    metric_kind = "GAUGE"
    value_type  = "DISTRIBUTION"
    unit        = "ms"
  }
  
  value_extractor = "EXTRACT(protoPayload.latencyMillis)"
}

# Log bucket for custom logs
resource "google_logging_project_bucket" "custom_logs" {
  name         = "custom-logs"
  location     = "US"
  description  = "Custom application logs"
  retention_days = 30
  
  lifecycle_state = "ACTIVE"
}

# Exclusions for noisy logs
resource "google_logging_project_exclusion" "health_checks" {
  name   = "exclude-health-checks"
  filter = <<-EOT
    resource.type="http_load_balancer" AND
    httpRequest.requestUrl:"/healthz"
  EOT
  
  description = "Exclude health check requests"
}

# Audit configuration
resource "google_project_service" "logging" {
  service = "logging.googleapis.com"
}

resource "google_project_iam_member" "logging_writer" {
  project = var.project
  role    = "roles/logging.logWriter"
  member  = "serviceAccount:${google_service_account.app.email}"
}

resource "google_project_iam_member" "logging_reader" {
  project = var.project
  role    = "roles/logging.viewer"
  member  = "user:analyst@company.com"
}
```

## CLI Examples

```bash
# Write log entry
gcloud logging write my-log "Test message" --severity=INFO

# Write with resource
gcloud logging write my-log "Application error" \
  --severity=ERROR \
  --log-name=my-app-log \
  --resource=gce_instance

# Read logs
gcloud logging read "severity>=ERROR" --limit=50
gcloud logging read "resource.type=cloud_run_revision" --limit=100

# Read from specific log
gcloud logging read "logName:my-app-log" --project=my-project

# Read using advanced filter
gcloud logging read \
  "resource.type=cloud_run_revision AND severity>=WARNING AND textPayload:exception" \
  --format="table(timestamp,severity,textPayload)"

# Write JSON log
echo '{"message":"Error occurred","code":500}' | \
gcloud logging write my-log --stdin --severity=ERROR

# List log entries
gcloud logging entries list --limit=100

# Delete log entries (by filter)
gcloud logging write my-old-log "" --severity=DEFAULT
# Note: You cannot delete individual entries, only whole logs

# Create sink
gcloud logging sinks create my-sink \
  storage.googleapis.com/my-bucket \
  --log-filter='severity>=WARNING'

# List sinks
gcloud logging sinks list

# Create log metric
gcloud logging metrics create error-metric \
  --description="Error count" \
  --log-filter='severity>=ERROR'

# List metrics
gcloud logging metrics list

# Write using Logger
logger -p user.error "My application error"
# This writes to syslog which Cloud Logging collects
```

## Troubleshooting

### Logs Not Appearing
```bash
# Check log name
gcloud logging read "logName:matches(^projects/.*/logs/my-app)"

# Verify resource type
gcloud logging read "resource.type=gce_instance"

# Check sink configuration
gcloud logging sinks describe my-sink

# Verify writer identity has permissions
gcloud logging sinks describe my-sink --format="yaml(writerIdentity)"
```

### Permission Issues
```bash
# Check log sink writer permissions
gcloud projects get-iam-policy my-project \
  --filter="bindings.members:serviceAccount:cloud-logs@google.com"

# Grant bucket access to sink writer
gcloud storage buckets add-iam-policy-binding gs://my-bucket \
  --member=serviceAccount:SINK_WRITER_SA \
  --role=roles/storage.objectCreator
```

## Interview Questions

**Q: What is the difference between log sinks and log buckets?**
A: Sinks route logs to destinations (Cloud Storage, BigQuery, Pub/Sub). Buckets store logs in Cloud Logging with configurable retention. Sinks can filter what to route; buckets define storage behavior.

**Q: How do you reduce logging costs?**
A: Exclude noisy logs (health checks, debug), use log buckets with shorter retention for non-production, limit audit log scope if possible, and aggregate before sinking.

**Q: What log types cannot be excluded?**
A: Admin Activity logs and Policy Denied logs cannot be excluded. Data Access logs can be filtered but generate costs.

**Q: How long are logs retained?**
A: Default retention is 30 days for most logs. Audit logs are retained 400 days. You can configure custom retention in log buckets.

**Q: What is a log-based metric?**
A: A metric derived from log entries using a filter. Types: counter (delta) for counting occurrences, and distribution for latency/size metrics.
