# Cloud Trace

## Overview

Google Cloud Trace is a distributed tracing system that collects latency data from applications and displays it in the Google Cloud Console. It helps identify performance bottlenecks by showing how requests flow through services. Cloud Trace supports OpenCensus, OpenTelemetry, and Zipkin instrumentation.

## When To Use

- Distributed application performance analysis
- Latency troubleshooting
- Service dependency mapping
- Request flow visualization
- Performance optimization

## When NOT To Use

- Real-time transaction monitoring
- High-frequency sampling needs (use custom solutions)
- Single monolithic application (use profiling instead)

## Core Concepts

### Trace
Complete end-to-end request path through services.

### Span
Individual operation within a trace (start/end time, metadata).

### Parent Span
Span that contains child spans (represents call hierarchy).

### Latency Distribution
P50, P90, P99 latency measurements per endpoint.

### Application Code (Python with OpenTelemetry)

```python
# requirements.txt
# opentelemetry-api
# opentelemetry-sdk
# opentelemetry-exporter-cloud-trace

from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.cloud_trace import CloudTraceSpanExporter
from opentelemetry.resources import Resource, SERVICE_NAME
from opentelemetry.sdk.resources import ATTR_SERVICE_NAME

# Initialize tracer
resource = Resource(attributes={
    ATTR_SERVICE_NAME: "my-service",
    "service.version": "1.0.0",
    "deployment.environment": "production"
})

provider = TracerProvider(resource=resource)
processor = BatchSpanProcessor(CloudTraceSpanExporter())
provider.add_span_processor(processor)
trace.set_tracer_provider(provider)

# Get tracer
tracer = trace.get_tracer(__name__)

# Example: HTTP request with tracing
import requests
from opentelemetry import propagate
from opentelemetry.trace.propagation.tracecontext import TraceContextTextMapPropagator

propagator = TraceContextTextMapPropagator()

def traced_request(url, method='GET', **kwargs):
    with tracer.start_as_current_span(f"http_{method.lower()}") as span:
        span.set_attribute("http.url", url)
        span.set_attribute("http.method", method)
        
        # Inject trace context into headers
        headers = kwargs.get('headers', {})
        propagator.inject(headers)
        kwargs['headers'] = headers
        
        with tracer.start_as_current_span("external_call"):
            try:
                response = requests.request(method, url, **kwargs)
                span.set_attribute("http.status_code", response.status_code)
                return response
            except Exception as e:
                span.set_attribute("error", True)
                span.record_exception(e)
                raise

# Example: Database query with tracing
def traced_db_query(query, params=None):
    with tracer.start_as_current_span("db_query") as span:
        span.set_attribute("db.statement", query[:200])  # Truncate for safety
        span.set_attribute("db.system", "postgresql")
        
        with tracer.start_as_current_span("execute"):
            result = db.execute(query, params)
        
        span.set_attribute("db.rows_affected", result.rowcount)
        return result

# Example: Business logic span
@tracer.start_as_current_span("process_order")
def process_order(order_id):
    span = trace.get_current_span()
    span.set_attribute("order.id", order_id)
    
    # Validate
    with tracer.start_as_current_span("validate_order"):
        validate_order(order_id)
    
    # Reserve inventory
    with tracer.start_as_current_span("reserve_inventory"):
        reserve_inventory(order_id)
    
    # Process payment
    with tracer.start_as_current_span("process_payment"):
        process_payment(order_id)
    
    # Send notification
    with tracer.start_as_current_span("send_notification"):
        send_notification(order_id)
```

### Node.js with OpenTelemetry

```javascript
// tracing.js
const { NodeSDK } = require('@opentelemetry/sdk-node');
const { CloudTraceSpanExporter } = require('@google-cloud/opentelemetry-cloud-trace-exporter');
const { HttpInstrumentation } = require('@opentelemetry/instrumentation-http');
const { ExpressInstrumentation } = require('@opentelemetry/instrumentation-express');
const { PgInstrumentation } = require('@opentelemetry/instrumentation-pg');

const sdk = new NodeSDK({
  traceExporter: new CloudTraceSpanExporter(),
  instrumentations: [
    new HttpInstrumentation(),
    new ExpressInstrumentation(),
    new PgInstrumentation()
  ],
  resource: {
    'service.name': 'my-service',
    'service.version': '1.0.0',
    'deployment.environment': process.env.NODE_ENV
  }
});

sdk.start();

// Graceful shutdown
process.on('SIGTERM', () => {
  sdk.shutdown()
    .then(() => console.log('Tracing shut down'))
    .catch((error) => console.error('Error shutting down tracing', error));
});
```

### Go with OpenCensus

```go
package main

import (
    "contrib.go.opencensus.io/exporter/stackdriver"
    "go.opencensus.io/trace"
)

func initTracer() {
    exporter, err := stackdriver.NewExporter(stackdriver.Options{
        ProjectID: "my-project",
    })
    if err != nil {
        log.Fatal(err)
    }
    trace.RegisterExporter(exporter)
    trace.ApplyConfig(trace.Config{
        DefaultSampler: trace.ProbabilitySampler(0.1), // 10% sampling
    })
}

// In your handlers
func handler(w http.ResponseWriter, r *http.Request) {
    ctx, span := trace.StartSpan(r.Context(), "handler")
    defer span.End()
    
    span.SetAttribute("http.method", r.Method)
    span.SetAttribute("http.url", r.URL.String())
    
    // Process request
    processRequest(ctx)
}
```

## Interview Questions

**Q: How does distributed tracing work?**
A: Each service injects trace context (trace ID, span ID) into request headers. Downstream services extract this context and create child spans. All spans are sent to a collector (Cloud Trace) where they're assembled into complete traces.

**Q: What is the difference between tracing and profiling?**
A: Tracing captures request flow across services with timing information. Profiling captures CPU and memory usage at the function level within a single service. Use tracing for latency issues, profiling for resource utilization.

**Q: How do you handle high-volume tracing?**
A: Use sampling to reduce trace volume. Head-based sampling (at trace start) or tail-based sampling (after completion) are common approaches. Cloud Trace automatically samples at 100% for first 100K spans/month.

**Q: What is trace context propagation?**
A: Trace context includes trace ID and span ID that travels with requests across service boundaries. W3C Trace Context standard defines headers (`traceparent`, `tracestate`) for context propagation.
