# Security Command Center

## Overview

Google Cloud Security Command Center (SCC) is a security posture management platform that provides threat detection, vulnerability identification, and security insights across GCP resources. It aggregates findings from various security scanners, integrates with SIEM tools, and supports compliance frameworks.

## When To Use

- Centralized security visibility
- Threat detection and alerting
- Compliance monitoring
- Vulnerability management
- Security posture tracking
- Event-driven security automation

## When NOT To Use

- Real-time network security (use Cloud Armor, Firewall)
- Identity management (use Cloud Identity)
- Encryption management (use Cloud KMS)

## Core Concepts

### Finding
Security issue identified by a scanner. Has severity, category, and resource.

### Security Health Analytics
Built-in checks for common security misconfigurations.

### Event Threat Detection
Detects threats in Cloud Logging (Enterprise tier).

### Security Score
Overall security posture score with category breakdowns.

### Asset Inventory
Discovery and tracking of GCP resources.

### Threat
Malicious activity detected in logs or network traffic.

### Security Automation Function (Node.js)

```javascript
// alerter.js - Cloud Function for processing security findings
exports.alerter = async (event, context) => {
    const pubsubMessage = event.data;
    const finding = JSON.parse(Buffer.from(pubsubMessage, 'base64').toString());
    
    const severity = finding.severity;
    const resource = finding.resourceName;
    const category = finding.category;
    
    console.log(`Processing finding: ${category} at ${resource} (${severity})`);
    
    if (severity === 'CRITICAL' || severity === 'HIGH') {
        await sendAlert(finding);
        
        if (category.includes('MALWARE')) {
            await isolateResource(finding);
        }
        
        if (category.includes('PRIVILEGE')) {
            await revokeAccess(finding);
        }
    }
    
    return { processed: true, finding: finding.category };
};

async function sendAlert(finding) {
    const message = {
        text: `Security Alert: ${finding.category}`,
        attachments: [{
            color: finding.severity === 'CRITICAL' ? 'danger' : 'warning',
            fields: [
                { title: 'Severity', value: finding.severity, short: true },
                { title: 'Resource', value: finding.resourceName, short: false },
                { title: 'Category', value: finding.category, short: true },
                { title: 'Description', value: finding.description, short: false }
            ]
        }]
    };
    
    await fetch(process.env.SLACK_WEBHOOK, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(message)
    });
}
```

```bash
gcloud scc securityscores list --organization=ORG_ID

gcloud scc assets list --organization=ORG_ID
gcloud scc assets list --organization=ORG_ID --asset-type=google.compute.Instance

# Get specific finding
gcloud scc findings describe FINDING_ID --organization=ORG_ID --source=SOURCE_ID

# Update finding status (mark as resolved)
gcloud scc findings update FINDING_ID \
  --organization=ORG_ID \
  --source=SOURCE_ID \
  --state=INACTIVE

# Create custom finding
gcloud scc findings create SOURCE_ID \
  --organization=ORG_ID \
  --category="CUSTOM_FINDING" \
  --description="Custom security finding" \
  --resource="//compute.googleapis.com/projects/my-project/zones/us-central1-a/instances/vm-1" \
  --severity=HIGH

# Enable Security Health Analytics
gcloud scc sources update SOURCE_ID \
  --organization=ORG_ID \
  --enable=False  # Enable via console typically

# Export findings
gcloud scc bigquery-exports create ORG_ID \
  --dataset=DATASET_ID \
  --filter="severity >= LOW"

# Set notification config
gcloud scc notification-configs create ORG_ID \
  --config-id=my-config \
  --pubsub-topic=projects/my-project/topics/my-topic \
  --filter="severity >= MEDIUM"
```

## Interview Questions

**Q: What is the difference between Standard and Enterprise SCC?**
A: Standard provides Security Health Analytics, Asset Discovery, and basic findings. Enterprise adds Event Threat Detection (real-time log analysis), Container Threat Detection, VM Threat Detection, and custom security analytics with longer retention.

**Q: How does Event Threat Detection work?**
A: ETD continuously analyzes Cloud Logging entries for malicious activity patterns. It uses predefined detection rules and ML to identify threats. When a threat is detected, it creates a finding with MITRE ATT&CK mapping.

**Q: What is a security finding?**
A: A finding represents a security issue identified by a scanner. It includes: severity, category, resource affected, timestamp, and recommended remediation. Findings can be marked active/inactive.

**Q: How do you automate response to findings?**
A: Create notification configs pointing to Pub/Sub. Cloud Functions or other services subscribe to process findings and take automated actions (isolate resources, revoke access, create tickets).

**Q: What is Security Health Analytics?**
A: SHA scans for common security misconfigurations: open ports, public bucket access, SQL public IP, IAM policy issues, and more. It runs continuously and creates findings when issues are detected.
