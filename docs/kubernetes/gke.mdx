# Google Kubernetes Engine (GKE)

## Overview

**Google Kubernetes Engine (GKE)** is Google's fully managed Kubernetes service that provides enterprise-grade container orchestration on Google Cloud. GKE handles control plane management, node provisioning, and infrastructure operations, allowing teams to focus on application workloads.

## When to Use

- **Enterprise workloads** requiring high availability
- **Cloud-native applications** on Google Cloud
- **Hybrid cloud** with GKE Enterprise
- **ML/AI workloads** with GPU node pools
- **Microservices** requiring managed infrastructure
- **Compliance requirements** with FIPS 140-2 validated modules

## When NOT to Use

- **Cost-sensitive projects** - EKS or AKS may be cheaper
- **Multi-cloud strategy** - consider cloud-agnostic solutions
- **Very small teams** - learning curve for GCP
- **Legacy workloads** - requires containerization effort

## Real Production Example

### Production GKE Cluster

```yaml
# Terraform for GKE cluster
# (Note: Terraform is typically used for production GKE)

# gke-cluster.tf
resource "google_container_cluster" "production" {
  name = "production-cluster"
  location = "us-central1"
  project = "my-project"
  
  # Release channel (Stable, Regular, Rapid)
  release_channel {
    channel = "Stable"
  }
  
  # Network
  network    = google_compute_network.vpc.name
  subnetwork = google_compute_subnetwork.subnet.name
  
  # IP allocation (VPC Native)
  ip_allocation_policy {
    cluster_ipv4_cidr_block  = "/14"
    services_ipv4_cidr_block = "/20"
    cluster_secondary_range_name  = "pods"
    services_secondary_range_name = "services"
  }
  
  # Private cluster
  private_cluster_config {
    enable_private_nodes    = true
    enable_private_endpoint = false
    master_ipv4_cidr_block  = "172.16.0.0/28"
  }
  
  # Master authorized networks
  master_authorized_networks_config {
    cidr_blocks {
      cidr_block   = "10.0.0.0/8"
      display_name = "VPN Network"
    }
  }
  
  # Node pools
  node_pool {
    name = "system-pool"
    node_count = 3
    
    node_config {
      machine_type = "n2-standard-4"
      disk_size_gb = 100
      disk_type    = "pd-standard"
      
      preemptible = false
      
      shielded_instance_config {
        enable_secure_boot = true
        enable_integrity_monitoring = true
      }
      
      labels = {
        "node-pool" = "system"
      }
    }
    
    management {
      auto_upgrade = true
      auto_repair  = true
    }
  }
  
  node_pool {
    name = "workload-pool"
    node_count = 5
    
    autoscaling {
      min_node_count = 3
      max_node_count = 20
    }
    
    node_config {
      machine_type = "n2-standard-8"
      disk_size_gb = 100
      disk_type    = "pd-standard"
      
      preemptible = false
      
      labels = {
        "node-pool" = "workload"
      }
      
      taint {
        key    = "workload-type"
        value  = "production"
        effect = "NoExecute"
      }
    }
    
    management {
      auto_upgrade = true
      auto_repair  = true
    }
  }
  
  # Security
  enable_legacy_abac         = false
  enable_integrity_monitoring = true
  enable_shielded_nodes       = true
  
  # Network policy
  network_policy {
    enabled  = true
    provider = "CALICO"
  }
  
  # Vertical Pod Autoscaling
  vertical_pod_autoscaling {
    enabled = true
  }
  
  # Workload Identity
  workload_identity_config {
    workload_pool = "${var.project}.svc.id.goog"
  }
  
  # Maintenance window
  maintenance_window {
    day_of_week = "SUNDAY"
    start_time  = "02:00"
  }
  
  # Logging and monitoring
  logging_config {
    component_config {
      enable_components = ["SYSTEM_COMPONENTS", "WORKLOADS"]
    }
  }
  
  monitoring_config {
    component_config {
      enable_components = ["SYSTEM_COMPONENTS", "WORKLOADS"]
    }
  }
}

# Node pool for GPUs
resource "google_container_node_pool" "gpu-pool" {
  name     = "gpu-pool"
  location = "us-central1"
  cluster  = google_container_cluster.production.name
  
  initial_node_count = 0
  
  autoscaling {
    min_node_count = 0
    max_node_count = 5
  }
  
  node_config {
    machine_type   = "a2-highgpu-1g"
    accelerator_count = 1
    accelerator_type  = "nvidia-tesla-t4"
    
    disk_size_gb = 100
    disk_type    = "pd-standard"
    
    preemptible = true
    
    labels = {
      "gpu" = "true"
    }
    
    taint {
      key    = "nvidia.com/gpu"
      value  = "present"
      effect = "NoSchedule"
    }
    
    labels = {
      "cloud.google.com/gke-accelerator" = "nvidia-tesla-t4"
    }
  }
}
```

### GKE Workload Identity

```yaml
# Bind Kubernetes service account to GCP service account
apiVersion: v1
kind: ServiceAccount
metadata:
  name: api-service-account
  namespace: production
  annotations:
    iam.gke.io/gcp-service-account: api-sa@my-project.iam.gserviceaccount.com

---
# RBAC for the Kubernetes service account
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: workload-identity-role
  namespace: production
rules:
- apiGroups: [""]
  resources: ["configmaps"]
  verbs: ["get", "list", "watch"]

---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: workload-identity-binding
  namespace: production
subjects:
- kind: ServiceAccount
  name: api-service-account
  namespace: production
roleRef:
  kind: Role
  name: workload-identity-role
  apiGroup: rbac.authorization.k8s.io

---
```

```yaml

metadata:
  name: api-ingress
  namespace: production
  annotations:
    kubernetes.io/ingress.class: "gce"
```

### GKE Autopilot (Serverless-like)

```yaml
# GKE Autopilot managed workloads
apiVersion: apps/v1
kind: Deployment
metadata:
  name: api-service
  namespace: production
spec:
  replicas: 3
  selector:
    matchLabels:
      app: api-service
  template:
    metadata:
      labels:
        app: api-service
    spec:
      # Autopilot handles node provisioning
      # Just set resource requests
      containers:
      - name: api
        image: gcr.io/myproject/api:v1.0
        resources:
          requests:
            cpu: "500m"
            memory: "512Mi"
          # Limits are optional in Autopilot
        readinessProbe:
          httpGet:
            path: /healthz/ready
            port: 8080
```

## Interview Questions

### Basic Questions

**Q: What is GKE?**
A: Google Kubernetes Engine (GKE) is Google's fully managed Kubernetes service that handles control plane management, node provisioning, and infrastructure operations on Google Cloud Platform.

**Q: What are the benefits of GKE over self-managed Kubernetes?**
A: GKE provides automatic control plane management, automatic upgrades, node auto-repair, integrated logging/monitoring, Workload Identity for GCP IAM, and managed add-ons like Ingress and Config Connector.

**Q: What is Workload Identity?**
A: Workload Identity allows Kubernetes service accounts to authenticate as GCP service accounts, enabling pods to access GCP APIs without storing service account keys.

### Intermediate Questions

**Q: What is the difference between GKE Standard and GKE Autopilot?**
A: Autopilot provisions and manages nodes automatically based on workload needs. Standard gives full control over node pools. Autopilot is serverless-like with simplified operations.

**Q: How does GKE networking work with VPC-native clusters?**
A: VPC-native clusters use alias IP ranges for pods and services, eliminating the need for additional routing tables. Pod IPs are automatically routable within the VPC.

**Q: What are GKE node pool upgrade strategies?**
A: Node pools can be upgraded using surge upgrades (surge count configurable) or blue-green upgrades (create new nodes, then delete old). GKE handles the upgrade with zero-downtime if PDBs are configured.

### Advanced Questions

**Q: How would you implement disaster recovery for GKE?**
A: Use etcd backups via Cloud Storage, regular application backups, cross-region cluster replication, or multi-cluster services. Test restore procedures regularly. Use Config Sync for GitOps.

**Q: How does GKE handle multi-tenancy?**
A: GKE supports namespace-based isolation, RBAC, NetworkPolicies, and node isolation with node pools. For stronger isolation, use GKE Enterprise with Policy Controller and Config Sync.
