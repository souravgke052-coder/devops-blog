# Kubernetes Labs

## Overview

Master Kubernetes through hands-on labs covering cluster operations, application deployment, networking, storage, and security. These labs use real kubectl commands and actual cluster configurations.

## Prerequisites

- kubectl installed and configured
- Access to a Kubernetes cluster (minikube, kind, or GKE/EKS/AKS)
- Docker or container runtime for building images

### Required Setup

```bash
# Verify kubectl installation
kubectl version --client
kubectl version --short

# Check cluster access
kubectl cluster-info
kubectl get nodes

# Configure kubectl context
kubectl config current-context
kubectl config get-contexts
```

## Lab 1: Pod and Deployment Management

**Objective**: Create, scale, and manage Kubernetes deployments
**Estimated Time**: 35 minutes
**Prerequisites**: Running Kubernetes cluster, kubectl configured

### Step-by-Step

#### 1. Create Your First Deployment

```bash
# Create a deployment using kubectl
kubectl create deployment nginx --image=nginx:latest --replicas=2

# Or create from YAML
cat > nginx-deployment.yaml << 'EOF'
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx
  labels:
    app: nginx
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
        resources:
          limits:
            memory: "128Mi"
            cpu: "500m"
          requests:
            memory: "64Mi"
            cpu: "250m"
EOF

kubectl apply -f nginx-deployment.yaml

# Check deployment status
kubectl get deployments
kubectl get pods
kubectl get replicasets
```

#### 2. Scale and Update Deployments

```bash
# Scale deployment
kubectl scale deployment nginx --replicas=5

# Scale based on load (requires metric-server)
kubectl autoscale deployment nginx --min=2 --max=10 --cpu-percent=70

# Update image version
kubectl set image deployment/nginx nginx=nginx:1.25-alpine

# Check rollout status
kubectl rollout status deployment/nginx

# View rollout history
kubectl rollout history deployment/nginx

# Undo rollout
kubectl rollout undo deployment/nginx

# Undo to specific revision
kubectl rollout undo deployment/nginx --to-revision=3
```

#### 3. Debug Pod Issues

```bash
# Get pod details
kubectl get pod nginx-5d59d6756c-klm9n -o wide
kubectl describe pod nginx-5d59d6756c-klm9n

# View pod logs
kubectl logs nginx-5d59d6756c-klm9n
kubectl logs nginx-5d59d6756c-klm9n --previous  # previous container logs

# Stream logs in real-time
kubectl logs -f nginx-5d59d6756c-klm9n

# Execute command in pod
kubectl exec nginx-5d59d6756c-klm9n -- ls /usr/share/nginx/html
kubectl exec -it nginx-5d59d6756c-klm9n -- /bin/sh

# Port forward for local debugging
kubectl port-forward nginx-5d59d6756c-klm9n 8080:80
```

#### 4. Resource Management

```bash
# Apply resource limits to running deployment
kubectl set resources deployment nginx \
    --limits=cpu=500m,memory=128Mi \
    --requests=cpu=250m,memory=64Mi

# Create deployment with resource quotas
cat > nginx-with-resources.yaml << 'EOF'
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-with-resources
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx-with-resources
  template:
    metadata:
      labels:
        app: nginx-with-resources
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
        resources:
          limits:
            memory: "256Mi"
            cpu: "500m"
          requests:
            memory: "128Mi"
            cpu: "100m"
        env:
        - name: NGINX_PORT
          value: "80"
        livenessProbe:
          httpGet:
            path: /
            port: 80
          initialDelaySeconds: 3
          periodSeconds: 3
        readinessProbe:
          httpGet:
            path: /
            port: 80
          initialDelaySeconds: 5
          periodSeconds: 5
EOF

kubectl apply -f nginx-with-resources.yaml
```

### Validation

```bash
# Verify deployment
kubectl get deployments -o wide
kubectl get pods -o wide
kubectl get replicasets

# Check pod resources
kubectl top pods
kubectl top nodes

# Verify probes are working
kubectl describe pod -l app=nginx-with-resources | grep -A5 "Liveness\|Readiness"
```

### Cleanup

```bash
kubectl delete deployment nginx
kubectl delete deployment nginx-with-resources
kubectl delete hpa nginx
rm nginx-deployment.yaml nginx-with-resources.yaml
```

## Lab 2: Services and Networking

**Objective**: Configure Kubernetes services, ingress controllers, and network policies
**Estimated Time**: 40 minutes
**Prerequisites**: Running cluster with deployments

### Step-by-Step

#### 1. Create ClusterIP Service

```bash
# Create deployment first
kubectl create deployment webapp --image=nginx:alpine --replicas=3

# Create ClusterIP service
cat > webapp-service.yaml << 'EOF'
apiVersion: v1
kind: Service
metadata:
  name: webapp-service
  labels:
    app: webapp
spec:
  type: ClusterIP
  selector:
    app: webapp
  ports:
  - port: 80
    targetPort: 80
    protocol: TCP
    name: http
  - port: 443
    targetPort: 443
    protocol: TCP
    name: https
EOF

kubectl apply -f webapp-service.yaml

# List services
kubectl get svc
kubectl get svc -o wide

# Test service connectivity (from another pod)
kubectl run test --rm -it --image=busybox --restart=Never -- \
    wget -qO- http://webapp-service
```

#### 2. Create NodePort and LoadBalancer Services

```bash
# NodePort service for external access
cat > nodeport-service.yaml << 'EOF'
apiVersion: v1
kind: Service
metadata:
  name: nodeport-service
spec:
  type: NodePort
  selector:
    app: webapp
  ports:
  - port: 80
    targetPort: 80
    nodePort: 30080
    protocol: TCP
    name: http
EOF

kubectl apply -f nodeport-service.yaml

# LoadBalancer service (cloud only)
cat > loadbalancer-service.yaml << 'EOF'
apiVersion: v1
kind: Service
metadata:
  name: webapp-lb
  annotations:
    cloud.google.com/backend-config: '{"default": "webapp-backend"}'
spec:
  type: LoadBalancer
  selector:
    app: webapp
  ports:
  - port: 80
    targetPort: 80
  sessionAffinity: ClientIP
  sessionAffinityConfig:
    clientIP:
      timeoutSeconds: 10800
EOF

kubectl apply -f loadbalancer-service.yaml

# Watch LoadBalancer provisioning
kubectl get svc webapp-lb -w
```

#### 3. Configure Ingress

```bash
# Install ingress controller (nginx)
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/controller-v1.8.0/deploy/static/provider/cloud/deploy.yaml

# Create ingress resource
cat > webapp-ingress.yaml << 'EOF'
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: webapp-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
    nginx.ingress.kubernetes.io/ssl-redirect: "false"
    cert-manager.io/cluster-issuer: "letsencrypt-prod"
spec:
  ingressClassName: nginx
  rules:
  - host: webapp.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: webapp-service
            port:
              number: 80
  tls:
  - hosts:
    - webapp.example.com
    secretName: webapp-tls
EOF

kubectl apply -f webapp-ingress.yaml

# List ingress
kubectl get ingress
kubectl describe ingress webapp-ingress
```

#### 4. Network Policies

```bash
# Create network policy to restrict traffic
cat > network-policy.yaml << 'EOF'
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: webapp-network-policy
spec:
  podSelector:
    matchLabels:
      app: webapp
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          role: frontend
    ports:
    - protocol: TCP
      port: 80
  egress:
  - to:
    - podSelector:
        matchLabels:
          role: database
    ports:
    - protocol: TCP
      port: 5432
  - to:
    - namespaceSelector:
        matchLabels:
          name: kube-system
    ports:
    - protocol: UDP
      port: 53
EOF

kubectl apply -f network-policy.yaml

# View network policies
kubectl get networkpolicies
kubectl describe networkpolicy webapp-network-policy
```

### Validation

```bash
# Test service connectivity
kubectl run curl-test --rm -it --image=curlimages/curl -- \
    curl http://webapp-service

# Test ingress (if configured)
curl -H "Host: webapp.example.com" http://INGRESS_IP/

# Verify endpoints
kubectl get endpoints webapp-service

# Check ingress controller logs
kubectl logs -n ingress-nginx deploy/ingress-nginx-controller
```

### Cleanup

```bash
kubectl delete -f webapp-service.yaml
kubectl delete -f nodeport-service.yaml
kubectl delete -f loadbalancer-service.yaml
kubectl delete -f webapp-ingress.yaml
kubectl delete -f network-policy.yaml
kubectl delete deployment webapp
```

## Lab 3: ConfigMaps and Secrets

**Objective**: Manage application configuration using ConfigMaps and Secrets
**Estimated Time**: 30 minutes
**Prerequisites**: Running cluster with kubectl access

### Step-by-Step

#### 1. Create ConfigMaps

```bash
# Create ConfigMap from literal values
kubectl create configmap app-config \
    --from-literal=DATABASE_HOST=postgres.default.svc \
    --from-literal=DATABASE_PORT=5432 \
    --from-literal=LOG_LEVEL=info

# Create ConfigMap from file
echo "DATABASE_URL=postgres://localhost:5432/mydb" > config.txt
kubectl create configmap file-config --from-file=config.txt

# Create ConfigMap from env file
cat > app.env << 'EOF'
DATABASE_HOST=postgres.default.svc
DATABASE_PORT=5432
DATABASE_NAME=myapp
LOG_LEVEL=debug
EOF

kubectl create configmap env-config --from-env-file=app.env

# Create ConfigMap from YAML
cat > configmap.yaml << 'EOF'
apiVersion: v1
kind: ConfigMap
metadata:
  name: nginx-config
data:
  nginx.conf: |
    server {
      listen 80;
      server_name localhost;
      location / {
        root /usr/share/nginx/html;
        index index.html;
      }
      location /health {
        access_log off;
        return 200 "healthy\n";
      }
    }
EOF

kubectl apply -f configmap.yaml

# List ConfigMaps
kubectl get configmaps
kubectl get configmap app-config -o yaml
```

#### 2. Create Secrets

```bash
# Create generic secret from literals
kubectl create secret generic db-credentials \
    --from-literal=username=appuser \
    --from-literal=password=SecurePassword123

# Create TLS secret
kubectl create secret tls nginx-tls \
    --cert=path/to/cert.pem \
    --key=path/to/key.pem

# Create registry secret
kubectl create secret docker-registry gcr-secret \
    --docker-server=gcr.io \
    --docker-username=_json_key \
    --docker-password="$(cat service-account.json)" \
    --docker-email=you@example.com

# Create secret from YAML
cat > secret.yaml << 'EOF'
apiVersion: v1
kind: Secret
metadata:
  name: api-keys
type: Opaque
data:
  STRIPE_API_KEY: c2stX19hcGlfa2V5X3Rlc3Q=
  AWS_ACCESS_KEY: YWtrX2FjY2Vzc19rZXk=
  AWS_SECRET_KEY: YWtrX3NlY3JldF9rZXk=
stringData:
  # These will be base64 encoded automatically
  PLAINTEXT_KEY: my-plaintext-value
EOF

kubectl apply -f secret.yaml

# List secrets (values hidden)
kubectl get secrets
kubectl describe secret db-credentials
```

#### 3. Use ConfigMaps and Secrets in Pods

```bash
# Create pod with ConfigMap and Secret
cat > pod-with-config.yaml << 'EOF'
apiVersion: v1
kind: Pod
metadata:
  name: app-pod
spec:
  containers:
  - name: app
    image: nginx:alpine
    env:
    - name: DATABASE_HOST
      valueFrom:
        configMapKeyRef:
          name: app-config
          key: DATABASE_HOST
    - name: DATABASE_PORT
      valueFrom:
        configMapKeyRef:
          name: app-config
          key: DATABASE_PORT
    - name: DB_USER
      valueFrom:
        secretKeyRef:
          name: db-credentials
          key: username
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: db-credentials
          key: password
    envFrom:
    - configMapRef:
        name: env-config
    volumeMounts:
    - name: nginx-config-volume
      mountPath: /etc/nginx/conf.d
      readOnly: true
  volumes:
  - name: nginx-config-volume
    configMap:
      name: nginx-config
EOF

kubectl apply -f pod-with-config.yaml

# Verify environment variables in pod
kubectl exec app-pod -- env | grep -E "DATABASE|DB_|LOG"
```

### Validation

```bash
# Verify ConfigMap is injected
kubectl exec app-pod -- cat /etc/nginx/conf.d/nginx.conf

# Verify environment variables
kubectl exec app-pod -- env

# Check if secrets are properly mounted (base64 encoded)
kubectl exec app-pod -- cat /secrets/db-credentials/username
kubectl exec app-pod -- cat /secrets/db-credentials/password
```

### Cleanup

```bash
kubectl delete pod app-pod
kubectl delete configmap app-config file-config env-config nginx-config
kubectl delete secret db-credentials api-keys nginx-tls gcr-secret
rm -f config.txt app.env configmap.yaml secret.yaml pod-with-config.yaml
```

## Lab 4: Persistent Storage

**Objective**: Configure persistent volumes, persistent volume claims, and storage classes
**Estimated Time**: 35 minutes
**Prerequisites**: Running cluster with storage support

### Step-by-Step

#### 1. Create PersistentVolume

```bash
# Create PersistentVolume
cat > pv.yaml << 'EOF'
apiVersion: v1
kind: PersistentVolume
metadata:
  name: pv-local
  labels:
    type: local
spec:
  capacity:
    storage: 1Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  storageClassName: local-storage
  hostPath:
    path: "/mnt/data"
EOF

kubectl apply -f pv.yaml

# List PVs
kubectl get pv
kubectl describe pv pv-local

# Create PVC
cat > pvc.yaml << 'EOF'
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: pvc-local
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 500Mi
  storageClassName: local-storage
  selector:
    matchLabels:
      type: local
EOF

kubectl apply -f pvc.yaml

# Check PVC status
kubectl get pvc
kubectl describe pvc pvc-local
```

#### 2. Use PVC in Pod

```bash
# Create pod with PVC
cat > pod-with-pvc.yaml << 'EOF'
apiVersion: v1
kind: Pod
metadata:
  name: app-with-storage
spec:
  containers:
  - name: app
    image: nginx:alpine
    volumeMounts:
    - name: app-storage
      mountPath: /usr/share/nginx/html
  volumes:
  - name: app-storage
    persistentVolumeClaim:
      claimName: pvc-local
EOF

kubectl apply -f pod-with-pvc.yaml

# Write data to volume
kubectl exec app-with-storage -- sh -c "echo 'Hello from PVC' > /usr/share/nginx/html/index.html"

# Verify data persists
kubectl exec app-with-storage -- cat /usr/share/nginx/html/index.html
```

#### 3. Storage Classes and Dynamic Provisioning

```bash
# List default storage classes
kubectl get storageclass
kubectl get storageclass -o wide

# Create custom StorageClass (for cloud providers)
cat > storageclass-gcp.yaml << 'EOF'
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: fast-disk
provisioner: pd.csi.storage.gke.io
parameters:
  type: pd-ssd
  fstype: ext4
reclaimPolicy: Delete
allowVolumeExpansion: true
volumeBindingMode: WaitForFirstConsumer
EOF

kubectl apply -f storageclass-gcp.yaml

# Create PVC with dynamic provisioning
cat > pvc-dynamic.yaml << 'EOF'
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: pvc-dynamic
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
  storageClassName: fast-disk
EOF

kubectl apply -f pvc-dynamic.yaml

# Watch PVC provisioning
kubectl get pvc -w
```

#### 4. StatefulSet with Volume Claims

```bash
# Create StatefulSet
cat > statefulset.yaml << 'EOF'
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: web
spec:
  serviceName: "web"
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: nginx
        image: nginx:alpine
        ports:
        - containerPort: 80
          name: web
        volumeMounts:
        - name: www
          mountPath: /usr/share/nginx/html
  volumeClaimTemplates:
  - metadata:
      name: www
    spec:
      accessModes: ["ReadWriteOnce"]
      storageClassName: fast-disk
      resources:
        requests:
          storage: 1Gi
EOF

kubectl apply -f statefulset.yaml

# Check StatefulSet
kubectl get statefulset
kubectl get pods -l app=web

# Each pod gets its own PVC
kubectl get pvc -l app=web
```

### Validation

```bash
# Verify PVC is bound
kubectl get pvc

# Check StatefulSet volumes
kubectl get pvc -l app=web

# Test data persistence
kubectl exec web-0 -- sh -c "echo 'Stateful Pod 0' > /usr/share/nginx/html/index.html"
kubectl exec web-1 -- sh -c "echo 'Stateful Pod 1' > /usr/share/nginx/html/index.html"
kubectl exec web-0 -- cat /usr/share/nginx/html/index.html
kubectl exec web-1 -- cat /usr/share/nginx/html/index.html

# Verify volumes are independent
kubectl exec web-0 -- ls -la /usr/share/nginx/html/
```

### Cleanup

```bash
kubectl delete -f statefulset.yaml
kubectl delete -f pod-with-pvc.yaml
kubectl delete -f pvc-dynamic.yaml
kubectl delete -f pvc.yaml
kubectl delete -f pv.yaml
kubectl delete -f storageclass-gcp.yaml
rm -f pv.yaml pvc.yaml pod-with-pvc.yaml pvc-dynamic.yaml statefulset.yaml storageclass-gcp.yaml
```

## Lab 5: Helm Charts and Package Management

**Objective**: Install, configure, and manage applications using Helm
**Estimated Time**: 35 minutes
**Prerequisites**: Helm 3 installed, cluster access

### Step-by-Step

#### 1. Helm Basics

```bash
# Verify Helm installation
helm version

# Add repositories
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo add grafana https://grafana.github.io/helm-charts

# Update repositories
helm repo update

# List repositories and charts
helm repo list
helm search repo bitnami
helm search repo nginx

# Get chart values
helm show values bitnami/nginx
```

#### 2. Install and Manage Charts

```bash
# Install chart
helm install my-nginx bitnami/nginx

# List releases
helm list
helm list -A

# Check release status
helm status my-nginx
helm get all my-nginx

# Upgrade release
helm upgrade my-nginx bitnami/nginx --set service.type=LoadBalancer

# Rollback release
helm history my-nginx
helm rollback my-nginx 1

# Uninstall release
helm uninstall my-nginx
```

#### 3. Custom Values and Configuration

```bash
# Create custom values file
cat > my-values.yaml << 'EOF'
replicaCount: 3

image:
  repository: nginx
  tag: 1.25-alpine

service:
  type: ClusterIP
  port: 80

resources:
  limits:
    memory: 256Mi
    cpu: 500m
  requests:
    memory: 128Mi
    cpu: 100m

autoscaling:
  enabled: true
  minReplicas: 2
  maxReplicas: 10
  targetCPU: 70
EOF

# Install with custom values
helm install myapp bitnami/nginx -f my-values.yaml

# Override specific values at runtime
helm upgrade myapp bitnami/nginx \
    -f my-values.yaml \
    --set replicaCount=5 \
    --set service.type=LoadBalancer

# Get current values
helm get values myapp
helm get values myapp -a  # all (including defaults)
```

#### 4. Create Your Own Helm Chart

```bash
# Create new chart
helm create my-chart

# View chart structure
ls -la my-chart/
ls -la my-chart/templates/

# Edit chart values
cat > my-chart/values.yaml << 'EOF'
replicaCount: 2

image:
  repository: nginx
  tag: alpine
  pullPolicy: IfNotPresent

service:
  type: ClusterIP
  port: 80

ingress:
  enabled: true
  className: nginx
  annotations:
    cert-manager.io/cluster-issuer: letsencrypt-prod
  hosts:
    - host: myapp.example.com
      paths:
        - path: /
          pathType: Prefix

resources:
  limits:
    memory: 256Mi
    cpu: 500m
  requests:
    memory: 128Mi
    cpu: 100m

autoscaling:
  enabled: true
  minReplicas: 2
  maxReplicas: 5
  targetCPU: 80
EOF

# Lint chart
helm lint my-chart

# Package chart
helm package my-chart

# Install from local chart
helm install local-app ./my-chart
```

### Validation

```bash
# Verify installation
helm list
kubectl get all -l app.kubernetes.io/managed-by=Helm

# Test application
curl http://my-nginx

# Check release history
helm history my-nginx

# Template rendering (dry-run)
helm template my-test ./my-chart -f my-values.yaml
helm install my-test ./my-chart -f my-values.yaml --dry-run --debug
```

### Cleanup

```bash
helm uninstall my-nginx
helm uninstall myapp
helm uninstall local-app
rm -f my-values.yaml my-chart.tgz
rm -rf my-chart
```
