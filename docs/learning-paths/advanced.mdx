# Senior DevOps / Platform Engineer

## Target Audience

This path is for:
- DevOps/SRE engineers with 3+ years experience
- Platform engineers building internal developer platforms
- Technical leads designing infrastructure
- Anyone who has completed the Intermediate Path

**You should know**: Kubernetes, Terraform, CI/CD, monitoring, cloud networking

**You don't need**: Leadership experience, multi-cloud architecture at scale

## Estimated Duration

| Learning Mode | Duration |
|---------------|----------|
| Part-time (15-20 hrs/week) | 20-24 weeks |
| Full-time (30-40 hrs/week) | 10-12 weeks |
| Accelerated (40+ hrs/week) | 6-8 weeks |

## Prerequisites

### Required Before Starting
- [ ] Production Kubernetes experience
- [ ] Infrastructure as Code with Terraform (production)
- [ ] CI/CD pipeline design
- [ ] Monitoring and alerting implementation
- [ ] Cloud networking (VPC, load balancers, DNS)
- [ ] [Completed Intermediate Path](/learning-paths/intermediate) or equivalent 3+ years experience

### Recommended
- Multi-cloud experience
- Leadership or mentoring experience
- Open source contributions

## Phase 2: Service Mesh and Advanced Networking

### Read
- [Service Mesh](/docs/networking/service-mesh)
- [Istio](/docs/mesh/istio)
- [Linkerd](/docs/mesh/linkerd)
- [mTLS and Service Identity](/docs/security/service-mesh)

### Practice
- Install and configure Istio or Linkerd
- Implement mTLS between services
- Configure traffic splitting for canary deployments
- Set up service mesh observability
- Implement circuit breaking and retries

### Validate
```bash
# Can you perform these tasks?
istioctl install --set profile=default
kubectl apply -f VirtualService.yaml
kubectl get destinationrules
istioctl proxy-config stats svc -n namespace
# Configure mTLS in STRICT mode
# Implement weighted traffic splitting
```

**Checkpoint**: Migrate a multi-service application to service mesh with mTLS and observability

**Time Estimate**: 3-4 weeks

### Validate
```bash

kubebuilder create API --group apps --version v1 --kind MyResource
make install && make run
kubectl get crds | grep myresources

```

**Checkpoint**: Build a production-ready Kubernetes Operator for custom resource management

**Time Estimate**: 4-5 weeks

## Portfolio Projects

Complete these projects to demonstrate senior-level skills:

### Project 1: Internal Developer Platform
Build a comprehensive IDP:
- Backstage service catalog
- Self-service deployment templates (CDE)
- Policy enforcement
- Security scanning integration
- Metrics and monitoring

**Deliverables**: Running IDP + architecture documentation + adoption runbook

### Project 2: Multi-Region Active-Active Deployment
Design active-active architecture:
- Multi-region Kubernetes
- Global load balancing
- Data replication strategy
- Automated failover
- RTO/RPO < 5 minutes

**Deliverables**: Architecture diagram + implementation code + DR playbook

### Project 3: Chaos Engineering Program
Establish chaos engineering practice:
- Experiment library
- Steady-state hypothesis framework
- Automation with litmus or chaos-mesh
- Integration with CI/CD
- Regular game days

**Deliverables**: Experiment library + automation + game day report
