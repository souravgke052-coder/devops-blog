# Reverse Proxy

## Overview

A reverse proxy sits in front of backend servers, intercepting client requests and forwarding them to appropriate backends. Unlike a forward proxy (which clients use to reach the internet), a reverse proxy is invisible to clients—they think they're talking directly to the origin server. Reverse proxies provide TLS termination, load balancing, caching, request transformation, and security controls.

NGINX and HAProxy are the workhorse reverse proxies for Linux environments. NGINX handles static content, TLS termination, and proxying. HAProxy excels at L4/L7 load balancing with sophisticated health checks. Both are battle-tested in production at massive scale.

This page covers reverse proxy configuration, request manipulation, WebSocket proxying, and gRPC reverse proxying.

## When To Use

- **TLS termination**: Decrypt once, forward encrypted to backends
- **Load balancing**: Distribute traffic across multiple backends
- **Caching**: Cache responses to reduce backend load
- **A/B testing**: Route traffic between versions
- **API gateway**: Transform requests, add authentication

## When NOT To Use

- **High-throughput low-latency**: Bypass proxy for extreme performance
- **Simple static files**: Serve directly from CDN or web server
- **WebSocket at scale**: Consider dedicated WebSocket infrastructure

## Core Concepts

### NGINX vs HAProxy

| Feature | NGINX | HAProxy |
|---------|-------|---------|
| Static content | Yes (native) | No (needs http-server) |
| L4 proxy | Yes (stream) | Yes (mode tcp) |
| L7 proxy | Yes | Yes |
| Health checks | Basic | Advanced (HTTP, TCP, script) |
| ACLs | Limited | Full ACL system |
| Performance | High | Very high |
| Configuration | `http { server {} }` | frontend/backend |

### Request Flow

```
Client (203.0.113.1)
    │ HTTPS request to api.example.com:443
    ▼
Reverse Proxy (NGINX)
    │ TLS termination, decrypt
    │ Check cache (if enabled)
    │ Apply rate limits, ACLs
    ▼
Backend Pool (10.0.0.5:8080, 10.0.0.6:8080)
    │ Load balancing (round robin, least_conn, etc.)
    ▼
Upstream Server (10.0.0.5:8080)
    │ Returns response
    ▼
Reverse Proxy
    │ Modify headers (add X-Real-IP)
    │ Cache response (if enabled)
    │ Compress (gzip)
    ▼
Client
```

### Basic NGINX Configuration

```nginx
http {
    # Upstream definition
    upstream api_backend {
        least_conn;
        server 10.0.0.5:8080 max_fails=3 fail_timeout=30s;
        server 10.0.0.6:8080 max_fails=3 fail_timeout=30s;
        keepalive 32;
    }
    
    server {
        listen 443 ssl http2;
        server_name api.example.com;
        
        ssl_certificate /etc/ssl/certs/api.example.com.crt;
        ssl_certificate_key /etc/ssl/certs/api.example.com.key;
        
        # Proxy configuration
        location /api/ {
            proxy_pass http://api_backend;
            proxy_http_version 1.1;
            
            # Headers
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header Connection "";
            
            # Timeouts
            proxy_connect_timeout 5s;
            proxy_send_timeout 60s;
            proxy_read_timeout 60s;
            
            # Buffers
            proxy_buffering on;
            proxy_buffer_size 4k;
            proxy_buffers 8 4k;
        }
    }
}
```

### HAProxy Configuration

```bash
# /etc/haproxy/haproxy.cfg
global
    log /dev/log local0
    maxconn 4096
    ssl-default-bind-ciphers ECDHE-ECDSA-AES128-GCM-SHA256:...
    ssl-default-bind-options no-sslv3 no-tls-tickets

defaults
    log global
    mode http
    option httplog
    option dontlognull
    timeout connect 5s
    timeout client 50s
    timeout server 50s

# Frontend
frontend https_front
    bind *:443 ssl crt /etc/ssl/certs/api.example.com.pem
    default_backend api_backend

# Backend
backend api_backend
    balance roundrobin
    option httpchk GET /health
    http-check expect status 200
    
    server api1 10.0.0.5:8080 check inter 3s fall 3 rise 2
    server api2 10.0.0.6:8080 check inter 3s fall 3 rise 2
```

## Real Production Example

### NGINX Reverse Proxy with Advanced Features

```nginx
http {
    # Rate limiting zones
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=100r/s;
    limit_req_zone $http_authorization zone=token_limit:10m rate=10r/s;
    
    # Connection limiting
    limit_conn_zone $binary_remote_addr zone=addr:10m;
    
    # Cache configuration
    proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=api_cache:10m 
                   max_size=1g inactive=60m use_temp_path=off;
    
    upstream api_backend {
        least_conn;
        server 10.0.0.5:8080 weight=3;
        server 10.0.0.6:8080 weight=3;
        keepalive 64;
    }
    
    server {
        listen 443 ssl http2;
        server_name api.example.com;
        
        ssl_certificate /etc/ssl/certs/api.example.com.crt;
        ssl_certificate_key /etc/ssl/certs/api.example.com.key;
        
        # Gzip compression
        gzip on;
        gzip_types application/json application/javascript text/css;
        gzip_min_length 1000;
        
        # API endpoints
        location /api/v1/ {
            # Rate limiting
            limit_req zone=api_limit burst=200 nodelay;
            limit_conn addr 10;
            
            # Caching (cache API responses)
            proxy_cache api_cache;
            proxy_cache_valid 200 60s;
            proxy_cache_key "$scheme$request_method$host$request_uri";
            proxy_cache_use_stale error timeout http_500;
            add_header X-Cache-Status $upstream_cache_status;
            
            # Proxy to backend
            proxy_pass http://api_backend;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
        
        # A/B testing (canary deployment)
        location /api/ab-test/ {
            # 10% to new version
            if ($cookie_ab_version = "B") {
                proxy_pass http://api_backend_v2;
                break;
            }
            proxy_pass http://api_backend_v1;
        }
        
        # WebSocket proxying
        location /ws/ {
            proxy_pass http://ws_backend;
            proxy_http_version 1.1;
            
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            
            proxy_read_timeout 86400;  # Long timeout for WS
        }
        
        # gRPC proxying
        location /grpc/ {
            grpc_pass grpc://grpc_backend;
            
            # gRPC specific headers
            grpc_set_header X-Real-IP $remote_addr;
        }
    }
}
```

    nginx.ingress.kubernetes.io/proxy-send-timeout: "60"
    nginx.ingress.kubernetes.io/affinity: "cookie"
    nginx.ingress.kubernetes.io/session-cookie-name: "route"
    nginx.ingress.kubernetes.io/session-cookie-expires: "172800"

  ingressClassName: nginx
  tls:
    - hosts:
        - api.example.com
      secretName: api-tls
  rules:
    - host: api.example.com
      http:
        paths:
          - path: /api/
            pathType: Prefix
            backend:
              service:
                name: api-service
                port:
                  number: 8080
          - path: /grpc/
            pathType: Prefix

## Security

### Reverse Proxy Security

```nginx
# Hide version info
server_tokens off;

# Security headers
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Strict-Transport-Security "max-age=31536000" always;

# IP allowlist (internal APIs)
location /internal/ {
    allow 10.0.0.0/8;
    allow 192.168.0.0/16;
    deny all;
    
    proxy_pass http://internal_backend;
}

# Buffer overflow protection
client_body_buffer_size 16k;
proxy_buffer_size 4k;
large_client_header_buffers 4 8k;
```

## Troubleshooting

### Common Reverse Proxy Issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| 502 Bad Gateway | Backend unreachable | Check backend, health |
| 504 Gateway Timeout | Backend slow | Increase timeouts |
| 400 Bad Request | Large headers | Increase large_client_header_buffers |
| 413 Request Entity Too Large | Body too large | Increase client_body_buffer_size |
| WebSocket failing | Missing Upgrade header | Add Upgrade/Connection headers |

## Interview Questions

**Q1: How do you proxy WebSocket connections in NGINX?**

A: WebSocket requires the Connection: Upgrade and Upgrade: websocket headers. In NGINX: `proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade";`. Also set long read timeout: `proxy_read_timeout 86400;` because WebSocket connections are long-lived. The same applies to HAProxy with `option http-server-close` and `http-request set-header Upgrade %[var(txn.upgrade)]` if needed.

**Q2: What's the difference between a forward proxy and a reverse proxy?**

A: Forward proxy: client-facing, clients use it to reach external services. Client knows they're using a proxy (proxy environment variable). Used for caching, filtering, anonymity. Reverse proxy: server-facing, clients think they're talking to the origin server. Client doesn't know about proxy. Used for TLS termination, load balancing, caching, protection. Forward proxy: "I'm the client, use this to reach the internet." Reverse proxy: "I'm the server, requests come through me to reach actual servers."

## Cheat Sheet

### NGINX Location Matching

| Syntax | Matches |
|--------|---------|
| `location /uri` | Prefix /uri |
| `location = /uri` | Exact /uri |
| `location ^~ /uri` | Prefix, stops regex |
| `location ~ /uri` | Case-sensitive regex |
| `location ~* /uri` | Case-insensitive regex |

### HAProxy ACLs

```
acl is_api path_beg /api/
acl is_internal src 10.0.0.0/8
acl auth_ok http_auth(totp_users)

use_backend api_backend if is_api
use_backend internal_backend if is_internal
```
