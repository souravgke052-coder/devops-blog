# GCP Automation

## Overview

GCP automation with Terraform enables programmatic infrastructure deployment, configuration management, and operational workflows. This guide covers GCP-specific patterns, native integrations, and production-grade automation strategies.

## When To Use

- **Infrastructure provisioning**: Create GCP resources via Terraform
- **Multi-environment deployments**: Dev/staging/prod with consistent config
- **GCP service automation**: Cloud SQL, GKE, Cloud Functions, etc.
- **Policy enforcement**: Enforce GCP organization policies
- **Compliance automation**: Apply security baselines automatically

## When NOT To Use

- **Real-time scaling**: Use native autoscaling or Kubernetes for sub-minute scale events
- **One-off commands**: Use `gcloud` CLI for ad-hoc operations
- **GKE workload management**: Use kubectl/Helm for application workloads
- **Secret rotation**: Use Secret Manager rotation features

## Core Concepts

### GCP Terraform Providers

```hcl
# Standard Google provider
provider "google" {
  project = var.project_id
  region  = var.region
}

# Beta features (preview)

```

### GCP Resource Hierarchy

```
Organization
└── Folder
    └── Project
        └── Resource (Compute, Storage, etc.)
```

### Key GCP Services in Terraform

| Service | Resource Type | Use Case |
|---------|--------------|----------|
| Compute | `google_compute_*` | VMs, networks, load balancers |
| GKE | `google_container_*` | Kubernetes clusters |
| Cloud SQL | `google_sql_database_instance` | Managed databases |
| Cloud Storage | `google_storage_bucket` | Object storage |
| Cloud Functions | `google_cloudfunctions2_function` | Serverless |
| Secret Manager | `google_secret_manager_secret` | Secrets storage |
| Cloud Build | `google_cloudbuild_trigger` | CI/CD triggers |
| Cloud Scheduler | `google_cloud_scheduler_job` | Scheduled jobs |

### VPC Network Module

```hcl
# modules/network/vpc/main.tf
variable "project_id" {
  description = "GCP project ID"
  type        = string
}

variable "environment" {
  description = "Environment name"
  type        = string
}

variable "vpc_cidr" {
  description = "VPC CIDR range"
  type        = string
  default     = "10.0.0.0/8"
}

variable "routing_mode" {
  description = "Routing mode: REGIONAL or GLOBAL"
  type        = string
  default     = "REGIONAL"
}

variable "mtu" {
  description = "MTU size"
  type        = number
  default     = 1460
}

variable "vpc_asn" {
  description = "BGP ASN for Cloud Router"
  type        = number
  default     = 65001
}

resource "google_compute_network" "vpc" {
  name                    = "${var.environment}-vpc"
  auto_create_subnetworks = false
  description             = "Custom VPC for ${var.environment} environment"
  
  routing_mode = var.routing_mode
  mtu         = var.mtu
  
  # Private Google Access enabled by default
  private_ip_google_access = true
  
  # BGP routing for Cloud Router (for HA/VM migration)
  dynamic "bgp" {
    for_each = var.routing_mode == "GLOBAL" ? [1] : []
    content {
      asn = var.vpc_asn
    }
  }
  
  lifecycle {
    prevent_destroy = true
  }
}

resource "google_compute_firewall" "allow_internal" {
  name        = "${var.environment}-allow-internal"
  network     = google_compute_network.vpc.name
  direction   = "INGRESS"
  priority    = 65534
  source_ranges = [var.vpc_cidr]
  
  allow {
    protocol = "tcp"
    ports    = ["0-65535"]
  }
  
  allow {
    protocol = "udp"
    ports    = ["0-65535"]
  }
  
  allow {
    protocol = "icmp"
  }
}

resource "google_compute_firewall" "allow_ssh" {
  name        = "${var.environment}-allow-ssh"
  network     = google_compute_network.vpc.name
  direction   = "INGRESS"
  priority    = 1000
  
  # Limit SSH access to VPN/Bastion only
  source_ranges = var.bastion_cidr
  
  allow {
    protocol = "tcp"
    ports    = ["22"]
  }
  
  target_tags = ["bastion", "admin"]
}

# Private Service Connect for managed services
resource "google_compute_global_address" "vpc_peering" {
  name         = "${var.environment}-vpc-peering"
  purpose      = "VPC_PEERING"
  address_type = "INTERNAL"
  prefix_length = 16
  network       = google_compute_network.vpc.id
}

resource "google_service_networking_connection" "vpc_peering" {
  network       = google_compute_network.vpc.id
  service       = "servicenetworking.googleapis.com"
  
  reserved_peering_ranges = [google_compute_global_address.vpc_peering.name]
}

output "vpc_id" {
  value = google_compute_network.vpc.id
}

output "vpc_name" {
  value = google_compute_network.vpc.name
}

output "vpc_self_link" {
  value = google_compute_network.vpc.self_link
}
```

  type = string

  type = string

  type = string

  type = string
}

  type = string
}

variable "db_password" {
  type      = string
  sensitive = true
}

variable "db_tier" {
  type    = string
  default = "db-custom-2-8192"
}

variable "db_disk_size" {
  type    = number
  default = 100
}

variable "vpc_id" {
  type = string
}

variable "authorized_cidrs" {
  type    = list(string)
  default = []
}

resource "google_sql_database_instance" "postgres" {
  name             = "${var.environment}-postgres"
  database_version = var.database_version
  region           = var.region
  
  deletion_protection = var.environment == "prod" ? true : false
  
  settings {
    tier              = var.db_tier
    region            = var.region
    availability_type = var.environment == "prod" ? "REGIONAL" : "ZONAL"
    
    # High availability for production
    # Regional instance provides automatic failover
    if var.environment == "prod" {
      availability_type = "REGIONAL"
    }
    
    # Disk configuration
    disk_size      = var.db_disk_size
    disk_type      = "PD_SSD"
    disk_autoresize = true
    
    # Backup settings
    backup_configuration {
      enabled                        = true
      start_time                     = "03:00"
      point_in_time_recovery_enabled = var.environment == "prod"
      
      backup_retention_settings {
        retained_backups = var.environment == "prod" ? 14 : 7
        retention_unit   = "COUNT"
      }
    }
    
    # IP configuration - private IP only
    ip_configuration {
      ipv4_enabled    = false
      private_network = var.vpc_id
      require_ssl     = true
      
      # Additional authorized networks
      dynamic "authorized_networks" {
        for_each = var.authorized_cidrs
        content {
          name  = "cidr-${index(var.authorized_cidrs, authorized_networks.value)}"
          value = authorized_networks.value
        }
      }
      
      # IAM authentication
      iam_authentication = true
    }
    
    # Maintenance window - avoid peak hours
    maintenance_window {
      day          = 0  # Sunday
      hour         = 4  # 4 AM
      update_track = "stable"
    }
    
    # Insights for query performance
    insights_config {
      query_insights_enabled  = true
      query_string_length     = 1024
      record_application_tags = true
      record_client_address   = false
    }
    
    # Database flags
    database_flags {
      name  = "max_connections"
      value = "200"
    }
    
    database_flags {
      name  = "log_min_duration_statement"
      value = "1000"  # Log queries taking > 1 second
    }
    
    database_flags {
      name  = "pg_stat_statements.track"
      value = "all"
    }
    
    # Collation for proper sorting
    database_flags {
      name  = "icu.case_sensitive_collation"
      value = "off"
    }
    
    # Enable PITR
    if var.environment == "prod" {
      settings {
        backup_configuration {
          point_in_time_recovery_enabled = true
        }
      }
    }
  }
  
  # Timeouts
  timeouts {
    create = "30m"
    update = "30m"
    delete = "30m"
  }
}

# modules/ci/cloudbuild-trigger/main.tf
variable "project_id" {
  type = string
}

variable "environment" {
  type = string
}

  type = string
}

  type    = string
  default = "main"
}

  type    = string
  default = "cloudbuild.yaml"
}

resource "google_cloudbuild_trigger" "terraform_plan" {
  name        = "${var.environment}-terraform-plan"
  description = "Trigger Terraform plan for ${var.environment}"
  
  github {
    owner = split("/", var.github_repo)[0]
    repo  = split("/", var.github_repo)[1]
    
    push {
      branch = var.github_branch
    }
  }
  
  substitutions = {
    _ENVIRONMENT = var.environment
    _REGION      = "us-central1"
  }
  
  filename = var.cloudbuild_yaml
  
  service_account = google_service_account.cloudbuild.email
}

resource "google_cloudbuild_trigger" "terraform_apply" {
  name        = "${var.environment}-terraform-apply"
  description = "Trigger Terraform apply for ${var.environment} (manual approval required)"
  
  github {
    owner = split("/", var.github_repo)[0]
    repo  = split("/", var.github_repo)[1]
    
    push {
      branch = "^main$"  # Only on main branch
    }
  }
  
  substitutions = {
    _ENVIRONMENT = var.environment
    _REGION      = "us-central1"
    _AUTO_APPLY  = "true"
  }
  
  filename = var.cloudbuild_yaml
  
  # Require manual approval for production
  approval_config {
    required = var.environment == "prod" ? true : false
  }
  
  service_account = google_service_account.cloudbuild.email

## Interview Questions

### Q1: How do you manage GCP projects with Terraform?
**A:** Use `google_project` resource to create projects, `google_project_service` to enable APIs, and organize with folders for hierarchy. Use separate state per environment.

### Q2: What is the difference between google and google-beta providers?
**A:** `google-beta` provides early access to GCP features before they're GA. Use `google` for stable features, `google-beta` for preview features with caution about breaking changes.

### Q3: How do you configure Private Google Access?
**A:** Set `private_ip_google_access = true` on VPC or subnetwork. Enables VMs without external IPs to reach Google APIs and services.

### Q4: How do you implement high availability in GCP with Terraform?
**A:** Use regional managed instance groups, regional Cloud SQL, multi-zonal GKE clusters, global load balancers, and Cloud Armor for DDoS protection.

### Q5: What is VPC Service Controls and how do you configure it?
**A:** Creates security perimeters around GCP resources. Use `google_access_context_manager_service_perimeter` to define restricted services and allowed resources.

### Q6: How do you manage secrets in GCP for Terraform?
**A:** Store in Secret Manager, access via `google_secret_manager_secret_version` data source. Never commit secrets to state or config files.

### Q7: What is Workload Identity and why use it?
**A:** Allows Kubernetes service accounts to impersonate GCP service accounts. Eliminates need for service account keys. Use `google_service_account_iam_member` with workload identity binding.

### Q8: How do you set up Cloud Build for Terraform automation?
**A:** Create Cloud Build triggers, service account with proper IAM, cloudbuild.yaml with terraform steps. Use substitutions for environment-specific values.

### Q9: What organization policies should you enforce?
**A:** `compute.disableSerialPortAccess`, `compute.requireOsLogin`, `storage.uniformBucketLevelAccess`, `iam.disableServiceAccountKeyCreation` for security hardening.

### Q10: How do you encrypt data in GCP?
**A:** Use Google-managed encryption (default), CMEK via `encryption_key` on resources, customer-supplied encryption keys (CSEK), or Cloud KMS for key management.
