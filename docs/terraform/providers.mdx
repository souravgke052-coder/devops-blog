# Terraform Providers

## Overview

Providers are plugins that Terraform uses to interact with cloud platforms, SaaS providers, and other APIs. Each provider exposes resources that map to infrastructure objects. Understanding provider configuration is essential for proper authentication and resource management.

## When To Use

- **Any infrastructure resource**: Providers are required for all Terraform operations
- **Multi-cloud deployments**: Configure multiple providers for AWS, GCP, Azure
- **Custom providers**: Use provider mirrors or private registries
- **Provider version management**: Pin versions for production stability

## When NOT To Use

- **Manual resources**: Providers manage all Terraform resources
- **Direct API calls**: Use providers instead of raw API calls

## Core Concepts

### Provider Architecture

```
Terraform
    ├── Core: Reads config, builds resource graph
    └── Providers: CRUD operations for resources
         ├── google (GCP resources)
         ├── aws (AWS resources)
         ├── azuread (Azure AD)
         └── ...
```

### Provider Configuration

```hcl
provider "google" {
  project = "my-project"
  region  = "us-central1"
}
```

### Implied Resource Type

```hcl
# Full form
resource "google_compute_instance" "example" {
  provider = google
  # ...
}

# Short form (uses default provider)
resource "google_compute_instance" "example" {
  # ...
}
```

## Real Production Example

### Multiple Provider Configuration

```hcl
# versions.tf
terraform {
  required_version = ">= 1.5.0"
  
  required_providers {
    google = {
      source  = "hashicorp/google"
      version = "~> 5.0"
    }
    google-beta = {
      source  = "hashicorp/google-beta"
      version = "~> 5.0"
    }
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
    kubernetes = {
      source  = "hashicorp/kubernetes"
      version = "~> 2.0"
    }
  }
}
```

### GCP Provider Configuration

```hcl
# providers.tf
provider "google" {
  project = var.project_id
  region  = var.region
  zone    = "${var.region}-a"
  
  # Service account impersonation (recommended)
  impersonate_service_account = var.terraform_service_account
  
  # Custom user agent
  user_agent_string = "terraform-production/1.0"
  
  # Retry configuration
  max_retries = 5
  
  # Default labels applied to all resources
  default_labels = {
    environment = var.environment
    managed_by  = "terraform"
  }
}

# Beta provider for preview features
provider "google-beta" {
  project = var.project_id
  region  = var.region
  zone    = "${var.region}-a"
  
  impersonate_service_account = var.terraform_service_account
}
```

### Multi-Project Provider Configuration

```hcl
# Provider for shared services project
provider "google" {
  alias = "shared_services"
  project = var.shared_services_project_id
  region  = var.region
}

# Provider for data project
provider "google" {
  alias = "data"
  project = var.data_project_id
  region  = var.region
}

# Provider for production
provider "google" {
  alias = "prod"
  project = var.prod_project_id
  region  = var.region
  
  # More restrictive for prod
  impersonate_service_account = var.prod_terraform_sa
}

# Usage with provider alias
module "shared_vpc" {
  source = "./modules/shared-vpc"
  
  providers = {
    google = google.shared_services
  }
}

module "data_storage" {
  source = "./modules/storage"
  
  providers = {
    google = google.data
  }
}
```

### AWS Provider Configuration

```hcl
provider "aws" {
  alias = "us_east_1"
  region = "us-east-1"

  provider = aws.us_west_2

```

## HCL Examples

### Provider Version Constraints

```hcl
# Pin to specific version
required_providers {
  google = {
    source  = "hashicorp/google"
    version = "5.0.0"
  }
}

# Use version >= (recommended for prod)
required_providers {
  google = {
    source  = "hashicorp/google"
    version = ">= 5.0.0"
  }
}

# Allow minor updates only
required_providers {
  google = {
    source  = "hashicorp/google"
    version = "~> 5.0"  # >= 5.0, < 6.0
  }
}

# Advanced: Multiple version constraints
required_providers {
  google = {
    source  = "hashicorp/google"
    version = {
      # Must have these features
      version = ">= 5.0"
      # Exclude specific buggy version
      not_contains = "5.0.1"
    }
  }
}
```

### Provider Alias and Default

```hcl
# Default provider (no alias)
provider "google" {
  project = "project-a"
  region  = "us-central1"
}

# Alias for different region
provider "google" {
  alias   = "europe"
  project = "project-a"
  region  = "europe-west1"
}

# Alias for different project
provider "google" {
  alias   = "secondary"
  project = "project-b"
  region  = "us-central1"
}

# Resource using non-default provider
resource "google_compute_instance" "europe_server" {
  provider = google.europe
  # ...
}
```

### Provider Installation

```hcl
# Use custom provider mirror
terraform {
  required_providers {
    mycorp = {
      source  = "mycorp/myprovider"
      version = "~> 1.0"
    }
  }
  
  provider_meta "mycorp" {
    # Custom metadata for provider
  }
}
```

### GKE Provider Configuration

```hcl
# GKE cluster provider
data "google_client_config" "default" {}

data "google_container_cluster" "primary" {
  name     = "my-cluster"
  location = var.region
}

provider "kubernetes" {
  host                   = "https://${data.google_container_cluster.primary.endpoint}"
  cluster_ca_certificate = base64decode(data.google_container_cluster.primary.master_auth[0].cluster_ca_certificate)
  token                  = data.google_client_config.default.access_token
}
```

## CLI Examples

### Provider Initialization

```bash
# Initialize providers (automatic with terraform init)
terraform init

# Initialize with provider upgrade
terraform init -upgrade

# Download specific provider version
terraform init -get-plugins=true

# Plugin cache
export TF_PLUGIN_CACHE_DIR=/opt/terraform/plugin-cache
terraform init
```

### Provider Locking

```bash
# Create provider lock files (hashes for security)
terraform providers lock -platform=darwin_amd64 -platform=linux_amd64

# Lock with custom mirrors
terraform providers lock \
  -platform=darwin_amd64 \
  -provider=registry.terraform.io/hashicorp/google=5.0.0

# Verify provider hashes
terraform providers mirror /path/to/mirror
```

### Provider Commands

```bash
# List required providers
terraform providers

# Show provider schema
terraform providers schema

# Print provider configuration
terraform providers config

# Provider-specific init
terraform init -plugin-dir=/opt/terraform/plugins
```

### Provider Upgrade

```bash
# Upgrade all providers to latest
terraform init -upgrade

# Upgrade specific provider
terraform init -upgrade -get-plugins=true

# Pin specific version
terraform init -get-plugins=false

# List available versions
terraform version
```

|---------|-------------|

| `terraform providers schema` | Show provider schemas |
| `terraform init -upgrade` | Upgrade providers |
| `terraform providers config` | Show provider config |

## Best Practices

### 1. Pin Provider Versions

```hcl
terraform {
  required_providers {
    google = {
      source  = "hashicorp/google"
      version = "~> 5.0"
    }
  }
}
```

### 2. Use Consistent Provider Source

```hcl
# Always use fully qualified source
required_providers {
  google = {
    source  = "hashicorp/google"  # Full source
    version = "~> 5.0"
  }
}

# NOT just "google"
```

### 3. Use Service Account Impersonation

```hcl
provider "google" {
  impersonate_service_account = var.terraform_service_account
}
```

### 4. Separate Provider Config Per Environment

```hcl
# environments/prod/providers.tf
provider "google" {
  project = "prod-project"
  # ...
}

# environments/dev/providers.tf
provider "google" {
  project = "dev-project"
  # ...
}
```

### 5. Use Default Labels

```hcl
provider "google" {
  default_labels = {
    environment = var.environment
    managed_by  = "terraform"
    cost_center = var.cost_center
  }
}
```

## Security

### Authentication Methods

#### 1. Application Default Credentials (ADC)

```bash
# gcloud auth application-default login
# Or use service account
gcloud auth activate-service-account --key-file=service-account.json

# Terraform automatically uses ADC
provider "google" {
  project = var.project_id
}
```

#### 2. Service Account Key File

```hcl
provider "google" {
  credentials = file("service-account.json")
  project     = var.project_id
}
```

#### 3. Impersonation (Recommended for Prod)

```hcl
provider "google" {
  project = var.project_id
  
  impersonate_service_account = "terraform@project.iam.gserviceaccount.com"
}
```

#### 4. Workload Identity (GKE)

```yaml
# Kubernetes service account annotation
apiVersion: v1
kind: ServiceAccount
metadata:
  name: terraform
  annotations:
    iam.gke.io/gcp-service-account: terraform@project.iam.gserviceaccount.com
```

```hcl
provider "google" {
  # No credentials needed when running in GKE with workload identity
}
```

### Provider Security

```bash
# Verify provider checksums
terraform providers lock

# Use only official HashiCorp providers
required_providers {
  google = {
    source  = "hashicorp/google"  # Official source
  }
}

# Avoid third-party providers without audit
```

## Troubleshooting

### Provider Initialization Failure

```bash
# Error: Provider not found
# Solution: Initialize with network access
terraform init

# Or use plugin directory
terraform init -plugin-dir=/path/to/plugins

# Mirror providers
terraform providers mirror /path/to/mirror
```

### Provider Version Conflict

```bash
# Error: Provider requirement conflict
# Solution: Update version constraints
terraform {
  required_providers {
    google = {
      source  = "hashicorp/google"
      version = "~> 5.0"  # Update constraint
    }
  }
}

# Or upgrade
terraform init -upgrade
```

### Authentication Failures

```bash
# Error: Request had insufficient authentication scopes
# Solution: Check ADC configuration
gcloud auth application-default login
gcloud auth list

# Or use service account with proper roles
gcloud iam service-accounts keys create key.json \
  --iam-account=terraform@project.iam.gserviceaccount.com

export GOOGLE_APPLICATION_CREDENTIALS=key.json
```

### Provider Schema Mismatch

```bash
# Error: Provider schema is outdated
# Solution: Reinitialize
rm -rf .terraform
terraform init

# Lock provider versions
terraform providers lock
```

### Missing Required Attribute

```bash
# Error: Missing required attribute
# Solution: Check provider documentation
# provider "google" requires project
provider "google" {
  project = var.project_id  # Add missing attribute
}
```

## Interview Questions

### Q1: What is a Terraform provider?
**A:** A plugin that interfaces with cloud APIs to create, read, update, and delete resources. Providers expose resource types that Terraform manages.

### Q2: How do providers differ from resources?
**A:** Providers are plugins that define how to interact with APIs. Resources are specific infrastructure objects managed by providers.

### Q3: What is the difference between `google` and `google-beta` providers?
**A:** `google-beta` provides access to GCP preview features and APIs. Use `google` for stable GA features.

### Q4: How do you manage provider versions?
**A:** Use `required_providers` block with version constraints. Pin versions in production. Use `~>` for minor version compatibility.

### Q5: What authentication methods does the GCP provider support?
**A:** ADC (default), service account keys, service account impersonation, workload identity federation.

### Q6: What is provider alias used for?
**A:** Configuring multiple provider instances for different regions, projects, or environments within the same configuration.

### Q7: How do you secure provider authentication?
**A:** Use service account impersonation with minimal IAM roles, enable Workload Identity for GKE, never commit credentials to version control.

### Q8: How do you upgrade providers safely?
**A:** Test in non-production first, review changelog, pin versions after testing, use `terraform init -upgrade`.

## Cheat Sheet

### Provider Configuration
```hcl
provider "PROVIDER" {
  # Provider-specific configuration
}

provider "PROVIDER" {
  alias   = "name"  # Multiple instances
  project = "xxx"
}

# Using aliased provider
resource "xxx_resource" "example" {
  provider = PROVIDER.name
}
```

### Version Constraints
```hcl
# Exact version
version = "5.0.0"

# Greater than or equal
version = ">= 5.0.0"

# Minor version compatibility (5.x)
version = "~> 5.0"

# Range
version = ">= 5.0, < 6.0"
```

### Required Providers Block
```hcl
terraform {
  required_providers {
    PROVIDER = {
      source  = "hashicorp/PROVIDER"
      version = "~> 1.0"
    }
  }
}
```
